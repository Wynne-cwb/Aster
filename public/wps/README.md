# Aster WPS 版 — 安装与真机验证指引

> ⚠️ **当前状态：投机性代码草稿，真机一次都没跑过**（v2.5 Phase 30 真机门未过，STATE.md 2026-06-29）。
> 三宿主代码已写完并通过代码侧验证（tsc / build / 1194 tests），但 **WPS 真机零验证**。
> 本目录是让你在 Windows WPS 真机上一次性验证全部的安装包 + 测试脚本。

## 这装的是什么

- **Aster WPS 版 = 三宿主全工具**：金山表格（ET）/ 金山文字（Word）/ 金山演示（WPP）**都**实现了 AI 读 + 写 + 撤销，工具集对齐 Office for Web 版（Excel ~20 工具 / Word ~14 / PPT ~17）。
- **没实现、不会出现的工具**（诚实收口，不裸奔）：AI 生图 / 图库检索插入 / 看图（视觉）/ 批量操作 / 冻结窗格——这些在 WPS 版**故意不暴露**，AI 不会调到，**它们不出现是正常的，不是 bug**。
- ⚠️ **全部代码是盲写**：WPS JSAPI（VBA 风格）的行为是据官方文档推断，没有真机校正过。**最可能出错的地方**：颜色（VBA 用 BGR 不是 #RRGGBB）、Excel 透视表/图表/条件格式、PPT 原生表格/连接线、Word 样式名（英文名在中文 WPS 上）/ 批注。这些正是这次要测出来的。

## 第 1 步：安装 Aster 产品（两条路，二选一）

### 路径 A：oem.ini + jsplugins.xml（WPS 专业版）

1. 找到 WPS 的 `oem.ini`（通常 `C:\Users\<你>\AppData\Roaming\kingsoft\office6\` 或安装根目录 `office6\cfgs\`）。
2. 加两行（与 publish 模式互斥）：
   ```ini
   JsApiPlugin=true
   JSPluginsServer=https://wynne-cwb.github.io/Aster/wps/jsplugins.xml
   ```
3. 完全退出并重启 WPS，打开金山表格 → 功能区出现「Aster」标签 →「打开 Aster」。

### 路径 B：publish.html 一键装（WPS 个人版 12.1.0.16910+，oem.ini 受限时用）

1. 浏览器打开 `https://wynne-cwb.github.io/Aster/wps/publish.html`。
2. 点加载项行的「安装」（首次弹"允许浏览器打开 WPS"→ 允许；按钮变「卸载」即成功）。
3. **publish 模式要求 `oem.ini` 里 `JsApiPlugin=false`**（与路径 A 互斥）。
4. 完全退出并重启 WPS，打开金山表格 → 「Aster」标签。

### 2-1 金山表格（ET）

先在空白表里填几行假数据（比如 A 列部门、B 列销售额，5~6 行带表头）。

| # | 对 AI 说 | 看什么 | 风险点 |
|---|---------|--------|--------|
| 1 | 读一下当前选中区域 / 列出所有工作表 | 返回值和表里一致？ | 读路径（Value2 标量/2D、Address） |
| 2 | 在 A1 写入 123 | 单元格真变？不静默无反应？ | 基础写 + 回读 |
| 3 | 把 A1:B1 加粗、底色设成蓝色 | 加粗对、**颜色对不对**？ | 🔴 **颜色 BGR**（蓝变红=字节序错） |
| 4 | 把这块数据按销售额降序排序 | 行序对？ | Sort 位置参 |
| 5 | 给销售额这列加个高于平均值标红的条件格式 | 规则生效？撤销能清掉？ | 🔴 FormatConditions OM |
| 6 | 把这块数据转成表格 / 合并 A1:C1 / 删除重复行 | 各自生效？ | ListObjects / Merge / RemoveDuplicates |
| 7 | 用这块数据建个透视表，行=部门，值=销售额求和 | **建出来没有**？ | 🔴🔴 PivotCaches（最可能整个不存在） |
| 8 | 插入一个柱状图 | 图出来没？ | 🔴 ChartObjects OM |
| 9 | **全部撤销** | 上面每步逆序还原回原值？ | inverse 全链路 |
