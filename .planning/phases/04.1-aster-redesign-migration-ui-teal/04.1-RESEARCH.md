## Summary

Phase 04.1 是一次完整的视觉系统迁移 + 局部结构调整，迁移对象是 `src/styles.css`（1488 行）和 `src/components/` 下所有 11 个组件（含子目录）。主要工作分三块：

**块 A — CSS token 重写：** 现有 `styles.css` 以紫靛蓝渐变 + 玻璃拟态为核心（`--brand-grad`、`--glass-bg`、backdrop-filter），新系统以 `.v-quiet.acc-teal` 为参考，提取 token 层到 `:root` + `[data-theme="light"]` + `[data-theme="dark"]`，变量命名全换成设计稿体系（`--accent`/`--surface`/`--text` 等）。token 不在 `.v-quiet` 类下——现有 codebase 用 `#root[data-theme]` 机制，继续保留，不引入 `.v-quiet` 父类（直接在 `:root` 和 `[data-theme]` 上定义 token）。

**块 B — 组件重皮 + 结构调整：** InputBar 是最复杂的改动（D-01/D-02）：吸收 SelectionPill（selpill 搬入 inputbar 顶部）并接管齿轮按钮（从 App.tsx topbar 移到 tools 行左下），移除 ContextCard 整个组件（退役）。ChatBubble 换新的 bubble-user/bubble-ai 形状（teal filled + 内高光）。AgentControlBar 去玻璃拟态换 quiet surface 风格。ErrorBubble 换成带 inset 3px stripe 的新形态。ToolResultCard 的折叠卡套用 writeback 范式（D-05）。

**块 C — 收尾（D-07）：** 重写 CLAUDE.md §UI、改 2 个记忆文件、标 01-UI-SPEC.md 为 superseded、跑 `/gsd-sketch-wrap-up` 固化 design skill。

**关键风险提示：** `aster.css` 的 `.v-quiet` 基类 `--accent` 是橙色 `#E64A19`，teal 颜色存放在 `.v-quiet.acc-teal` 修饰类中（第 1755 行）。落地时必须引用 `.acc-teal` 覆盖块而非 `.v-quiet` 基础块的 accent 值。

**Primary recommendation:** 先写新 token 层（`:root` + light/dark），运行 `npm run build` 验 bundle 体积不超 70KB，再按「selpill/InputBar → ChatBubble → Onboarding → Settings → AgentControlBar/ErrorBubble/ToolCards」顺序逐组件重皮，边改边用浏览器截图自查。

### 新增（本 phase 字体）

| 字体 | Google Fonts URL | Weight 子集 | 使用场景 |
|------|-----------------|------------|---------|
| Inter | `css2?family=Inter:wght@400;500;600` | 400/500/600 | 正文拉丁字符、西文数字、按钮文案 |
| JetBrains Mono | `css2?family=JetBrains+Mono:wght@400;500` | 400/500 | 时间戳、模型 ID、URL、code block、error `.code` 代号 |

**字体加载策略（index.html 追加）：**
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  rel="stylesheet"
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&family=JetBrains+Mono:wght@400;500&family=Noto+Sans+SC:wght@400;500;700&display=swap"
/>
```
可合并成一条 Google Fonts URL，减少请求数。700 weight 只在 Noto Sans SC 需要（标题）；Inter + JetBrains Mono 不需要 700。

**Bundle 影响：** 字体文件不进 JS bundle（外链 Google Fonts），对 NFR-02 ~70KB JS 预算无影响。移除玻璃拟态（backdrop-filter + --glass-bg 相关 CSS）会减少 CSS 体积，净效果是 CSS 略微变小。

### Anti-Patterns to Avoid

- **不要在 `.v-quiet` 类下定义 token：** 现有 codebase `#root` 上没有 `.v-quiet` 类，直接用 `:root` + `[data-theme]` 选择器。
- **不要硬编码 `#009887`：** 所有颜色必须走 `var(--accent)`，保证 dark theme 自动切换。
- **不要在 JSX 里用 `style={{color: '#xxx'}}`：** 所有颜色走 CSS 变量，不散落硬编码 hex。
- **不要移除 lingui `<Trans>` 标记：** 重皮时所有中文字符串必须保留 i18n 标记，不能用裸字符串字面量替换。
- **不要在 InputBar 渲染 ContextCard：** D-02 已锁定退役，selpill 承担所有选区信息。
- **不要删 `backdrop-filter` 后忘删 `-webkit-backdrop-filter`：** 两个都要清。
- **不要给 give-up 红卡加假撤销按钮：** D-06 明确诚实禁用。

## Component-by-Component Migration Notes

### App.tsx — 结构调整（D-01/D-02）

**变化类型：** 结构调整（非纯重皮）

当前结构：`aster-key-hint` → `aster-topbar`（ContextCard + 齿轮按钮）→ `aster-shell__agent-bar` → `aster-chat` → `InputBar` → 设置覆盖层 → Onboarding

迁移后结构：`pane-banner`（key 缺失）→ `chat-scroll` → `inputbar-wrap` → 设置覆盖层 → Onboarding modal

## Common Pitfalls

### Pitfall 1: 误用 `.v-quiet` 基类 accent（橙色）当 teal

**What goes wrong:** 开发者提取 aster.css 的 `.v-quiet` 块得到 `--accent: #E64A19`（橙色），写入 styles.css，导致所有 accent 色变橙。
**Why it happens:** `aster.css` 设计了多种 accent 变体；teal 不是基础默认值，而是 `.acc-teal` 修饰类（第 1755 行）。
**How to avoid:** 提取时明确取 `.v-quiet.acc-teal` 块（`#009887` light / `#4FC9B8` dark），而非 `.v-quiet` 基础块。
**Warning signs:** 发送按钮、selpill 背景、focus ring 呈橙色。

### Pitfall 2: `.v-quiet` 父类选择器污染

**What goes wrong:** 把 aster.css 的 `.v-quiet { }` 整段 token 照搬，导致 CSS 只在 `.v-quiet` 父类节点下生效，但现有 codebase 的 `#root` 上没有 `.v-quiet` 类。
**Why it happens:** proto-app.jsx 用 `<div className="pane v-quiet acc-teal">` 包裹；现有 codebase 用 `<div class="aster-shell">`。
**How to avoid:** token 定义在 `:root` + `[data-theme="light|dark"]` 选择器，不包在 `.v-quiet` 下。组件样式仍可参照 aster.css，但选择器前缀适配现有结构。

### Pitfall 3: lingui 标记丢失

**What goes wrong:** 重皮时直接复制 proto-app.jsx 的中文字面量字符串（JSX `{「输入消息…」}` 形式），不加 `<Trans>` / `t\`\`` macro 包裹。
**Why it happens:** 设计稿是 JSX 裸字符串，无 lingui。
**How to avoid:** 所有新增/改动的可见中文字符串必须走 `<Trans>` 或 `useLingui().t\`\``，与现有组件保持一致（可参考 InputBar.tsx 现有写法）。
**Warning signs:** `npm run extract` 后出现新的未标记字符串 warning。

### Pitfall 4: `main.tsx` 主题设置点不兼容新 token 选择器

**What goes wrong:** 新 styles.css 用 `[data-theme="light"]` 定义 token，但 `main.tsx` 设置的是 `container.dataset.theme = "light"`（即 `#root[data-theme="light"]`），CSS 选择器匹配的是 `[data-theme="light"]`——这是 **OK 的**（`#root` 自身上有 `data-theme`），只要选择器写的是 `[data-theme="light"]` 而非 `body[data-theme]` 即可。
**Verification:** 确认新 styles.css 中 token 定义用 `[data-theme="light"]` 不写成 `html[data-theme]` 或 `.v-quiet[data-theme]`。

### Pitfall 5: backdrop-filter 残留导致白色光晕

**What goes wrong:** 删了 `--glass-bg` 背景色但没删 `backdrop-filter: blur(14px)` 声明，在浅色主题下出现意外模糊效果。
**How to avoid:** 删除所有 `backdrop-filter` 和 `-webkit-backdrop-filter` 声明，连同 `--glass-bg` / `--shadow-glass` / `--glow` 变量一起清理。

### Pitfall 6: ContextCard 退役后 App.tsx 仍有 import

**What goes wrong:** 删除 ContextCard 后 App.tsx 有 `import ContextCard from './components/ContextCard'`，TypeScript 编译报错。
**How to avoid:** 同步删除 App.tsx 的 import + JSX 渲染行 + 相关 `SettingsIcon` import（齿轮从 App.tsx 移走后 App.tsx 不再需要 SettingsIcon）。

### Pitfall 7: Office SVG 图标放错位置

**What goes wrong:** onboarding step2 的 `office-ppt.svg` 等图标需要通过 `import.meta.env.BASE_URL` 拼路径（同 aster-logo.png 一样），但如果错误地用 ES import 方式引入，在 manifest sideload 下路径会不对。
**How to avoid:** 放 `public/assets/` 目录，通过 `${import.meta.env.BASE_URL}assets/office-ppt.svg` 引用（同现有 `icon-80.png` 的方式）。

### Pitfall 8: dark theme token 名引用错误

**What goes wrong:** dark theme 下 `--accent-on` 是深色（`#0E0E10`），在气泡内写 `color: var(--accent-on)` 期望在 dark 下显示白色，但实际是深色。
**Why it happens:** dark theme 的 `--accent` 是浅色 teal，对应 `--accent-on` 是深色，这与 light 相反（light `--accent-on = #FFFFFF`）。
**How to avoid:** `--accent-on` 是「accent 色上的文字」，始终与 `--accent` 对比度合格。不需要特殊处理，由 token 体系保证。

## Open Questions

1. **`pillVisible` 的显示条件**
   - What we know: proto-app.jsx 用 `state.pillVisible` 全局 flag 控制 selpill 是否渲染；现有 SelectionPill 根据 `ctx !== ''`（有选区时显示）隐式控制。
   - What's unclear: 迁移后 selpill 是否也用「无选区时隐藏整行」逻辑，还是始终渲染（like current SelectionPill 的「始终渲染，无选区时灰态」逻辑）？
   - Recommendation: 保持现有逻辑（无选区时 selpill 行不渲染，有选区时渲染）。不引入新的 `pillVisible` zustand 字段（避免越位改功能，D-04 约束）。

2. **主题设置节点兼容性**
   - What we know: `main.tsx` 在 `container.dataset.theme = resolveHostTheme()` 处设置 `#root` 的 data-theme；新 styles.css token 用 `[data-theme="light|dark"]` 选择器。
   - What's unclear: 是否需要 `#root[data-theme]` 更具体的选择器来防止嵌套冲突？
   - Recommendation: 使用 `[data-theme="light"]` / `[data-theme="dark"]`（不加 `#root` 前缀），与设计包约定一致，且嵌套在 `#root` 内的所有元素都能通过 cascade 继承 token 值。

3. **Office SVG 图标授权**
   - What we know: 设计包 `src/icons/` 下有 `office-ppt.svg`/`office-excel.svg`/`office-word.svg`（「由用户提供」，README 注明为微软官方 logo）。
   - What's unclear: 在开源仓库中使用这些图标是否合规？
   - Recommendation: onboarding step2 的宿主图标可用文字标签（PPT/Excel/Word）+ 对应 Lucide 图标（如 `fileText`/`chartBar`/`type`）代替，避免版权风险。如用户确认可用官方图标，才放入 public/。
