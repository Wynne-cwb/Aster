# Phase 04.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> This is a **visual migration** (CSS token rewrite + component reskin + D-01/D-02 structural change) with **no new logic**. Validation is therefore weighted toward (a) light-theme pixel fidelity across PPT/Excel/Word, (b) dark-theme self-check + 1-host spot check, (c) regression guards that no functional capability is lost, (d) bundle budget (NFR-02), (e) i18n coverage.

## Sampling Rate

- **After every task commit:** Run `npm test -- --run` (quick, <30s)

- **Before `/gsd-verify-work`:** Full suite green + bundle ≤ 70KB gzipped verified
- **Phase gate (D-08):** Full suite green + light 三宿主 (PPT/Excel/Word) 真机 UAT pass + dark theme Claude browser self-check + 1-host dark spot check
