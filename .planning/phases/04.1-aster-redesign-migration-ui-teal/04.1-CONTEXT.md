**Gathered:** 2026-05-29

<domain>
## Phase Boundary

把现有 codebase（Vite + React 19 + TS + 自写 CSS）的视觉系统，从当前的「紫靛蓝品牌渐变 + 玻璃拟态」迁移到设计包定稿的「单一 teal `#009887` + 暖白底 `#FAFAF8` + 克制（quiet）、无玻璃拟态、无渐变」语言。

具体交付：
1. 重写 `src/styles.css` token 层（移植设计包 `aster.css` 的 `.v-quiet` token）
2. 重皮所有现有组件到新 teal 语言（像素级还原设计稿 light theme）
3. 落设计稿的核心**结构调整**（见 D-01/D-02）——本 phase 同时改结构，不只是换皮
4. 按新 teal 语言**补设计**设计包空白的 agent 运行时面（控制条 / read 折叠卡 / give-up 红卡 / ErrorBubble / 5秒卡住入口 / 步骤文案）的**视觉**
5. 收尾：更新 CLAUDE.md §UI + 改记忆 + 标 01-UI-SPEC.md 过时 + 固化 project design skill

**这只换皮肤/调结构布局，不加新功能能力。** 新功能（DiffLogPanel、写回多动作卡、killer-scenario chips）= Phase 5/6。

<decisions>
## Implementation Decisions

### 迁移范围边界（Area ①）
- **D-01:** 本 phase = **重皮 + 结构调整一起做**，组件树不冻结。除换 token/重皮外，同时落设计稿核心布局：选区上下文折进 InputBar 顶部 selpill、齿轮移到输入框底部 tools 行（左下）。理由：避免「先重皮再改结构」把组件摸两遍；交付后是完整新设计而非半迁移态。
- **D-02:** **移除顶栏 ContextCard**（Phase 4 CR-01「第 N 张幻灯片」上下文卡整条移除），选区上下文（宿主 + 选区描述 + CR-01 幻灯片号文案）改由 InputBar 顶部 `selpill` 承载 + eye 开关。**功能不丢，只是位置迁移**——CR-01 的幻灯片号信息进 selpill 文案。`src/components/ContextCard.tsx` 退役/合并进 InputBar selpill。
- **D-03:** **写回多动作聚合卡推 Phase 5**（DiffLogPanel 职责，且现在还没 write tool，无内容可渲染）；**empty-state killer-scenario chips 推 Phase 6**。本 phase empty-state **只做壳**：logo pulse 动画 + 标题「从你正在做的东西开始」+ 副文案，**不做会跳场景的假 suggestion chips**（诚实禁用原则）。

### Agent 运行时面设计深度（Area ②）
- **D-04:** 设计包空白的 in-flight 面（AgentControlBar / give-up 红卡 / 5秒卡住入口 / read 折叠卡 / 步骤差异化文案）= **视觉按新 teal 克制语言重做，交互骨架不动**。Phase 3/4 已真机验证过的交互全保留：pause/abort/step counter、5秒安抚文案、三态（读/思考/写）文案逻辑——只换视觉，不重建逻辑（迁移 phase 不越位改功能）。
- **D-05:** read 步骤折叠卡（Phase 4 D-01：`role=tool` 默认折一行）**套用设计稿写回卡的折叠范式**（chevron + 单行 target ellipsis + 展开 body），让 read / write 两类步骤卡视觉同源一致。Phase 4 D-01 本就选「默认折一行」，范式天然匹配。
- **D-06:** ErrorBubble **重皮成设计稿 ErrorBubble 定稿形态**：与 bubble-ai 同形状（左下角 4px）、左 3px accent stripe（`inset 3px 0 0 var(--error)`）、红字 head（600）+ alertTriangle 13px + mono `.code` 代号 + 可选红色下划线 cta。**Agent gave up 红卡套同一视觉范式**。诚实禁用不变：give-up 红卡不放假撤销按钮（undo 是 Phase 5），点了只出占位 toast。错误代号沿用 `src/errors/index.ts` 8 类（设计稿 ERROR_CATALOG 对账一致）。

### 收尾 & 验证范围（Area ③）
- **D-07:** 配套 doc/记忆/skill 更新**全部进本 phase**，作为收尾 plan：
  - 重写 `CLAUDE.md` §UI/Components + §Conventions「UI 设计系统」→ teal 克制 / 无玻璃 / 无渐变（推翻旧「渐变 accent + 玻璃拟态」约定）
  - 改记忆 `feedback_beauty_over_fluent.md`（根原则「美观优先、自写 CSS、不回 Fluent」保留，具体风格从「渐变+玻璃」更新为「teal 克制」）+ 更新 `project_aster_ui_redesign.md`（从「待迁移」→「已落地」）
  - 标 `.planning/phases/01-foundation/01-UI-SPEC.md` 过时（加 superseded 标记，指向本 phase + 设计包）
  - 跑 `/gsd-sketch-wrap-up` 把 teal 设计系统固化成 project design skill（供 Phase 5/6 新 UI 直接消费，零返工）
  - 理由：不更新 CLAUDE.md/记忆，下个 phase 的 Claude 会被旧「玻璃拟态」约定误导，与实际 styles.css 矛盾（[[recurring_failure_add_gate]] 思路：结构性消除矛盾源）
- **D-08:** 真机 UAT 深度：**light 三宿主（PPT/Excel/Word）真机 UAT 必做**（迁移影响全部可见 UI，light 是主要风险面）；**dark theme 以 Claude 浏览器自查为主**（dark token 已定但从未 QA，Claude 逐屏浏览器截图自查），**真机只抽查 1 个宿主的 dark**。理由：真机 UAT 成本在用户身上（[[self_run_spikes]]：Claude 自跑能跑的别让用户重跑），dark 风险低于 light。

### Claude's Discretion
- **Token 命名策略**（用户未选此灰区，授权 Claude 定）：**采用设计稿 `aster.css` 的变量名**（`--accent` / `--surface` / `--surface-2/3` / `--text` / `--text-2` / `--text-3` / `--border` / `--bubble-ai-bg` 等），替换现有 `--brand` / `--text-1` 等旧名。理由：让未来设计交付与 codebase 1:1 对得上（设计包/design skill 都用这套名），一次迁移到位胜过命名长期漂移；代价是要改每个组件里的 var/className 引用，但本 phase 反正要重皮全部组件，顺手做。主题选择器从现有 `[data-theme="light|dark"]` 适配设计稿 `.v-quiet` + `.v-quiet[data-theme="dark"]`——保留现有 `data-theme` 机制（main.tsx 读 officeTheme），不强引 `.v-quiet` 变体类。
- **字体引入**：index.html 现仅加载 Noto Sans SC（Google Fonts）。补加 **Inter**（正文拉丁 + 西文数字）+ **JetBrains Mono**（时间戳/模型 ID/URL/code/代号）两个 family，沿用现有 Google Fonts `display=swap` + 系统中文栈兜底机制（已在 Google Fonts 上，不破纯静态约定）。具体 weight 子集由 Claude 按设计稿用量裁剪（控 bundle/字体体积）。
- 动效 token（`--dur-fast/base/slow`、`--ease-out/spring`、empty-mark pulse 4s）、阴影 token、radius/spacing scale 全照设计稿定稿值移植；`prefers-reduced-motion` 全局降级沿用。
- 组件拆分优先级（按设计包建议）：`<Bubble variant>` / `<InputBar>`（含 selpill/textarea/tools row）/ `<Modal>`（onboarding）/ Settings 行 + 滑入 overlay——具体落到现有组件文件的重构由 planner 定。

</decisions>

<specifics>
## Specific Ideas

- 用户全程取「推荐」默认，决策气质：克制、信任设计包、不重建已验证的东西。planner 任何 UI 取舍前先想：这是「让迁移后是完整新设计」还是「让用户做更多真机活」——前者做满，后者能 Claude 自查就别推给用户（[[self_run_spikes]] / [[feedback_beauty_over_fluent]]）。
- 「诚实禁用」贯穿（CLAUDE.md §UI）：empty-state 不做假 chips（D-03）、give-up 红卡不放假撤销键（D-06）、附件按钮 disabled 态保留「即将开放」。
- 像素级还原指 **light theme**；dark 是「token 已定、靠 Claude 浏览器自查 + 真机抽查」级别（D-08），不要求 dark 也像素级背书。

</specifics>
