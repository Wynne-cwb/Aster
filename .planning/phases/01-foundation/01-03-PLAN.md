import type { DocumentAdapter, SelectionContext, AdapterCapabilities, InsertableContent } from './DocumentAdapter';
import { UnsupportedOperationError, HostApiError } from '../errors';

SelectionContext 变体（plan 02 定义）：
- { kind: 'ppt'; slideIndex: number; slideCount: number }
- { kind: 'excel'; address: string }
- { kind: 'word'; charCount: number }
- { kind: 'none' }   // D-16 无选中

DocumentAdapter 方法：
- getSelection(): Promise&lt;SelectionContext&gt;
- onSelectionChanged(callback: () =&gt; void): () =&gt; void   // 返回解绑函数（D-13）
- capabilities(): AdapterCapabilities
- insert(content: InsertableContent): Promise&lt;void&gt;
</interfaces>

<office_js_api_notes>
<!-- Level 1 验证：context7 CLI/MCP 本环境不可用；以下 API 模式来自 PATTERNS.md 枚举 + Office.js 已知契约。
     executor 实现时须依赖 tsconfig 的 @types/office-js 类型核对签名（types: ["office-js"] 已配）。 -->

- PPT getSelection：PowerPoint.run(async (ctx) =&gt; { const sel = ctx.presentation.getSelectedSlides(); sel.load('items'); const all = ctx.presentation.slides; all.load('items'); await ctx.sync(); ... })
  - slideIndex = 第一个选中 slide 的 1-based 序号（「第 N 张」）；slideCount = 全部 slide 数。
  - PPT-05 守则（Phase 4 严格强制，骨架最好遵循）：getSelectedSlides() 结果按 .index 排序后再用（绕 Web 反序 bug #3618）。
  - PPT selection 事件：Office.context.document.addHandlerAsync(Office.EventType.DocumentSelectionChanged, handler)；解绑 removeHandlerAsync(Office.EventType.DocumentSelectionChanged, { handler })。
- Excel getSelection：Excel.run(async (ctx) =&gt; { const r = ctx.workbook.getSelectedRange(); r.load('address'); await ctx.sync(); return r.address; })
  - selection 事件：ctx.workbook.worksheets.getActiveWorksheet().onSelectionChanged.add(handler) —— 返回 EventHandlerResult，.remove() 解绑（在 Excel.run 内 remove）。
- Word getSelection：Word.run(async (ctx) =&gt; { const sel = ctx.document.getSelection(); sel.load('text'); await ctx.sync(); return sel.text.length; }) —— charCount = sel.text.length。
  - selection 事件：ctx.document.onSelectionChanged.add(handler) —— EventHandlerResult .remove() 解绑。
- 共性：Office.js 事件 add/remove 异步。onSelectionChanged 对外返回的解绑函数内部封装 remove。无选中时 getSelection 返回 { kind: 'none' }（D-16），不抛错。Office.js 异常包成 HostApiError（plan 02）。
- API 混用守则（spike #5022）：不在同一路径混用 setSelectedDataAsync 与 PowerPoint.run（写进 PptAdapter 顶部注释）。
</office_js_api_notes>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Information Disclosure | getSelection 读取文档内容 | mitigate | Phase 1 adapter 仅读取元数据（slide 序号 / range 地址 / 字符数），**不读取选区正文内容**（Word charCount 用 text.length，不留存 text）。正文读取与发往 Provider 是 Phase 2 行为，受 KEY-03 隐私告知约束；本阶段不为其埋下默认外发通道。 |
| T-01-07 | Tampering | onSelectionChanged 事件解绑 | mitigate | 解绑函数必须真实移除宿主 handler（防止 Task Pane 隐藏后事件继续触发——与 Phase 2 PROV-07 visibilitychange abort 同源约束）；smoke test 虽不验真机解绑，但接口契约强制返回解绑函数。 |
| T-01-08 | Denial of Service | insert 桩 | accept | Phase 1 insert 抛 UnsupportedOperationError，不执行任何写回——无 DoS 面。Phase 4-6 写回时再评估。 |
</threat_model>
