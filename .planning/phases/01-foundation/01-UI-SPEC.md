# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase intent (CONTEXT D-06):** This phase ships a *visual shell*, not feature logic. The full 350px three-section Task Pane skeleton (top context card + middle chat stream + bottom input + upload icon + Provider dropdown) is drawn so AC1's "350px three-section look" lands in one step. Phase 2/3 fill logic only — they do not change layout.
>
> **Hard constraint (CLAUDE.md):** Component library is **Fluent UI React v9** only. shadcn / AntD / MUI are explicitly rejected — native Office visual parity is load-bearing for the BYO-key trust value prop. The shadcn initialization gate is therefore **not applicable** to this project.
>
> **Token-first rule:** All spacing, type, and color values below are expressed as **Fluent v9 design tokens** (`@fluentui/tokens`), not raw px/hex. Hardcoding raw values would break the Office host light / dark / high-contrast theming that this phase must follow (CONTEXT references CLAUDE.md theming). The px/hex shown is the *light-theme resolved value* for reference only — the executor must reference the **token**, never the literal.

> ## ⚠️ POST-IMPLEMENTATION REVISION — 2026-05-27（本契约下方多数内容已被取代）
>
> 本 UI-SPEC 当初以「**Fluent UI v9 only + 原生 Office 观感 + token-first**」为前提编写并签核。
> **用户在真机看过后判定该观感太丑/格格不入，明确以美观为第一位，推翻了这条美学约束。**
>
> 现已落地的实现与下方契约的差异：
> - **不再用 Fluent UI v9**（`@fluentui/react-components`/`@fluentui/react-icons` 已从依赖移除）。改为自写 CSS 设计系统 `src/styles.css`（CSS 变量驱动 light/dark）+ 内联 SVG 图标 `src/components/icons.tsx`（Lucide 风/ISC）。
> - **风格** = 柔和品牌渐变 + 玻璃拟态；品牌渐变只作 accent（发送键 / logo 光晕 / focus ring），**不做大面积渐变带/header**（与 Office chrome 冲突）。
> - **不再画品牌 header**：Office 面板已自带「Aster」原生标题；顶部行 = 上下文卡 + 中性设置齿轮（Provider 等配置归设置）。
> - **输入区** = 统一输入容器（WeChat 范式：工具左下 / 发送右下），取代原「Provider 下拉 + 上传 + 输入框 + 发送」四件套；Provider 不在输入栏暴露切换器（仅 AiHubMix / DeepSeek，归设置）。
> - 字体 = Noto Sans SC + 系统中文栈兜底（不再用 Segoe UI/host font）。
> - 主题随 Office 宿主仍成立（`data-theme`），i18n / 选区逻辑 / 诚实禁用语义保留。
>
> **未变的硬约束**：无后台、纯静态、Office webview、Key 存浏览器、JS 体积预算（移除 Fluent 后实测 ~63–68KB gzip）。
>
> **新的 UI 风格 source of truth**：`CLAUDE.md` §Conventions「UI 设计系统」+ `src/styles.css`。下方「Design System / Typography / Color / Component Inventory」各表中涉及 Fluent token 的描述均按上述实现理解，**仅作历史留存**。

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Fluent v9 ships `spacingHorizontal*` / `spacingVertical*` tokens. All values below are multiples of 4. Executor must reference the token, not the px.

| Token (Fluent v9) | Value | Usage in this phase |
|-------------------|-------|---------------------|
| `spacingHorizontalXS` / `spacingVerticalXS` | 4px | Icon-to-label gaps inside context card; inline chip padding |
| `spacingHorizontalS` / `spacingVerticalS` | 8px | Gap between input field and send button; compact stack gaps |
| `spacingHorizontalM` / `spacingVerticalM` | 16px | Default padding inside the three sections (context card, chat area, input bar); section content inset |
| `spacingHorizontalL` / `spacingVerticalL` | 24px | Vertical padding around the empty-state block (centered in chat area) |
| `spacingHorizontalXL` | 32px | Reserved — empty-state breathing room on taller viewports |

Exceptions:
- **Task Pane fixed width: 350px** (PANE-01 / AC1) — this is a layout dimension, not a spacing token. Set as the pane container min-width; pane is user-resizable (PANE-01 "可调").
- **Ribbon button touch/click targets** are owned by the Office host ribbon, not by our CSS — no spacing token applies.
- Section heights: top context card = content-height (auto), middle chat stream = `flex: 1` (fills remaining), bottom input bar = content-height (auto). The three-section split is flex-column, not fixed pixel heights.

**Source:** PANE-01 (350px three-section), CONTEXT D-06, Fluent v9 token set.

## Typography

All roles map to Fluent v9 typography tokens (`Text` component `size`/`weight` props or `tokens.fontSize*`). Two weights only: `regular` (400) and `semibold` (600). 4 sizes declared.

| Role | Fluent token | Size | Weight | Line Height | Used for |
|------|--------------|------|--------|-------------|----------|
| Body | `fontSizeBase300` | 14px | regular (400) | `lineHeightBase300` ≈ 1.43 | Chat message text, input field text, context card value text |
| Label | `fontSizeBase200` | 12px | regular (400) | `lineHeightBase200` ≈ 1.33 | Disabled-placeholder hints ("Provider（即将开放）"), context card caption, tooltips |
| Heading | `fontSizeBase400` | 16px | semibold (600) | `lineHeightBase400` ≈ 1.5 | Empty-state heading「开始使用 Aster」 |
| Display | `fontSizeBase500` | 20px | semibold (600) | `lineHeightBase500` ≈ 1.5 | Reserved — not used in Phase 1 shell; available for Phase 2 onboarding titles |

Rules:
- Body line-height target ≈ 1.5 is satisfied by Fluent's paired `lineHeightBase*` tokens — do not override.
- Never use `weight="bold"` (700) — only `regular` and `semibold` are in the contract.
- All visible strings are wrapped in Lingui macros and ship zh-CN only (FOUND-08 / CONTEXT D-17). Typography applies to the rendered Chinese strings.

**Source:** spike `main.tsx` (already uses `Text size={600} weight="semibold"`), CLAUDE.md (Fluent v9 typography), Fluent v9 token set.

## Color

Driven entirely by Fluent v9 semantic tokens so Office host light / dark / HC themes apply for free. The hex shown is the **light-theme resolved value for reference only** — executor references the token.

| Role | Fluent token | Light value (ref) | Usage |
|------|--------------|-------------------|-------|
| Dominant (60%) | `colorNeutralBackground1` | `#ffffff` | Task Pane base surface, chat stream background |
| Secondary (30%) | `colorNeutralBackground2` / `colorNeutralBackground3` | `#fafafa` / `#f5f5f5` | Top context card surface, bottom input-bar surface (the two framing sections that "sandwich" the chat stream) |
| Accent (10%) | `colorBrandBackground` | `#0f6cbd` | Send button primary color slot only (see reserved list) |
| Neutral text | `colorNeutralForeground1` / `colorNeutralForeground3` | `#242424` / `#616161` | Body text / secondary captions + disabled-placeholder hints |
| Destructive | `colorPaletteRedForeground1` | `#b10e1c` | **Not used in Phase 1** — no destructive actions exist in the shell (see Copywriting). Reserved for Phase 2 settings/key-delete. |

**Accent reserved for (explicit list):**
1. The bottom **「发送」(Send) button** `appearance="primary"` color slot — it keeps the brand color even while disabled in Phase 1 (disabled state dims it via Fluent's own disabled token, but the color slot stays brand).
2. The **context-card refresh "active" indicator** — a brief brand-colored pulse/tint on the card when `onSelectionChanged` fires and the value updates, signalling "this is live" (CONTEXT D-12). Subtle, token-driven.

Accent is NOT used for: disabled Provider dropdown, disabled upload icon, empty-state text, context-card text, chat bubbles, borders, or any other interactive element. Everything else is neutral.

**Rationale:** In a shell where nearly every control is a disabled placeholder, restraint on accent is what keeps it reading as "native Office, intentionally not-yet-active" rather than broken. Confirmed by user 2026-05-27.

**Source:** User confirmation 2026-05-27; Fluent v9 semantic token set; CONTEXT D-07/D-08/D-12.

## Copywriting Contract

All strings zh-CN, wrapped in Lingui macros (CONTEXT D-17). Tone: AntD-style Chinese copy patterns rendered in Fluent components (CLAUDE.md guidance — Chinese-audience benefit is copy + tone, not foreign components).

| Element | Copy (zh-CN) | Notes |
|---------|--------------|-------|
| Primary CTA | 发送 | Send button label. **Disabled in Phase 1** (bottom input is inert — CONTEXT D-08). Visible, brand color slot, disabled state. |
| Empty state heading | 开始使用 Aster | `fontSizeBase400` semibold, centered in chat stream |
| Empty state body | 配置 Provider 后即可开始对话 | `fontSizeBase300` regular, neutral-foreground-3, centered under heading. Honest capability boundary (CONTEXT D-08). |
| 空状态用法提示小标题 | 试试这些 | `fontSizeBase200` neutral-foreground-3，居于示例芯片之上（FOUND-10 用法引导）。 |
| Ribbon 入口按钮 | 打开 Aster | 三宿主统一入口 Label（manifest `Btn.Aster.Open.Label`），点击打开 Task Pane。 |
| Provider dropdown placeholder | Provider（即将开放） | Disabled dropdown placeholder text (CONTEXT D-07). |
| Upload icon tooltip | 文件上传即将开放 | Fluent `Tooltip` on the disabled upload icon (CONTEXT D-07). |
| Context card — no selection | 未选中内容 | Short version, 350px-width-friendly (CONTEXT D-16). No error thrown. |
| Context card — PPT | 第 N 张 slide | Live value from `getSelection()` (CONTEXT D-14 / ROADMAP SC3). N = slide index. |
| Context card — Excel | 选中区域 {address} | Live value, e.g. "选中区域 A1:C10" (CONTEXT D-14 / ROADMAP SC3). |
| Context card — Word | 选中 {n} 字 | Live value, selected-text character count (CONTEXT D-14 / ROADMAP SC3). |
| Input field placeholder | 输入消息… | Bottom input placeholder; field is disabled in Phase 1 (CONTEXT D-08). |

**Destructive actions:** None in Phase 1. The shell has no delete / clear / irreversible actions — every control is either a disabled placeholder or a read-only live display. No destructive-confirmation copy required. (Destructive UX arrives in Phase 2 with Key management.)

**Source:** User confirmation 2026-05-27; CONTEXT D-07/D-08/D-14/D-16/D-17; ROADMAP SC3.

## Component Inventory (Fluent v9)

What the executor builds this phase, mapped to Fluent v9 components. All from `@fluentui/react-components` via specific imports (no barrel).

| Section | Component | Fluent v9 element | State in Phase 1 |
|---------|-----------|-------------------|------------------|
| Root | App shell | `FluentProvider` (host-themed) + flex-column 350px container | Live (theming + Office.onReady) |
| Top | Context card | `Card` / styled surface + `Text` | **Live** — reflects `getSelection()` via `onSelectionChanged` (D-12/D-13) |
| Middle | Chat stream | scrollable flex region + empty-state block (`Text` heading + body) | Empty-state only (no messages yet) |
| Bottom | Input bar | `Textarea`/`Input` + upload `Button`(icon) + `Dropdown` (Provider) + `Button` (Send, primary) | All **disabled placeholders** (D-07/D-08), upload icon + dropdown have explanatory tooltips/placeholder |
| Ribbon | 1 个统一 Aster 入口按钮/宿主（共 3 个） | Office manifest `<Control>` (`Action=ShowTaskpane`) | Click → open Task Pane only (D-09/D-10/D-11/FOUND-10) |

**Ribbon 入口（最终决策 — FOUND-10）：**
- 每宿主单一入口，统一文案「打开 Aster」、共用 Aster 图标组（D-10）。
- Office for Web 技术上每宿主需各 1 个 Control 定义（id 不同：Aster.Open / AsterXL.Open / AsterWD.Open），但 Label/Tip/图标完全统一 —— 概念上是「一个 Aster 入口」。
- Ribbon 不再承载功能动作；原本想靠功能按钮承载的功能入口改由 Task Pane 空状态用法提示承载（见 Copywriting「空状态用法提示」）。

入口文案 zh-CN Lingui-wrapped。所有 AI 能力均在 Task Pane 内触发。

**Source:** PANE-01, CONTEXT D-06/D-07/D-08/D-09/D-10/D-11/D-12/D-13, FOUND-10.
