# Phase 1 έΑΦ UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase intent (CONTEXT D-06):** This phase ships a *visual shell*, not feature logic. The full 350px three-section Task Pane skeleton (top context card + middle chat stream + bottom input + upload icon + Provider dropdown) is drawn so AC1's "350px three-section look" lands in one step. Phase 2/3 fill logic only έΑΦ they do not change layout.
>
> **Hard constraint (CLAUDE.md):** Component library is **Fluent UI React v9** only. shadcn / AntD / MUI are explicitly rejected έΑΦ native Office visual parity is load-bearing for the BYO-key trust value prop. The shadcn initialization gate is therefore **not applicable** to this project.
>
> **Token-first rule:** All spacing, type, and color values below are expressed as **Fluent v9 design tokens** (`@fluentui/tokens`), not raw px/hex. Hardcoding raw values would break the Office host light / dark / high-contrast theming that this phase must follow (CONTEXT references CLAUDE.md theming). The px/hex shown is the *light-theme resolved value* for reference only έΑΦ the executor must reference the **token**, never the literal.

## Spacing Scale

Fluent v9 ships `spacingHorizontal*` / `spacingVertical*` tokens. All values below are multiples of 4. Executor must reference the token, not the px.

| Token (Fluent v9) | Value | Usage in this phase |
|-------------------|-------|---------------------|
| `spacingHorizontalXS` / `spacingVerticalXS` | 4px | Icon-to-label gaps inside context card; inline chip padding |
| `spacingHorizontalS` / `spacingVerticalS` | 8px | Gap between input field and send button; compact stack gaps |
| `spacingHorizontalM` / `spacingVerticalM` | 16px | Default padding inside the three sections (context card, chat area, input bar); section content inset |
| `spacingHorizontalL` / `spacingVerticalL` | 24px | Vertical padding around the empty-state block (centered in chat area) |
| `spacingHorizontalXL` | 32px | Reserved έΑΦ empty-state breathing room on taller viewports |

Exceptions:
- **Task Pane fixed width: 350px** (PANE-01 / AC1) έΑΦ this is a layout dimension, not a spacing token. Set as the pane container min-width; pane is user-resizable (PANE-01 "ίΠψϋ░Δ").
- **Ribbon button touch/click targets** are owned by the Office host ribbon, not by our CSS έΑΦ no spacing token applies.
- Section heights: top context card = content-height (auto), middle chat stream = `flex: 1` (fills remaining), bottom input bar = content-height (auto). The three-section split is flex-column, not fixed pixel heights.

**Source:** PANE-01 (350px three-section), CONTEXT D-06, Fluent v9 token set.

## Typography

All roles map to Fluent v9 typography tokens (`Text` component `size`/`weight` props or `tokens.fontSize*`). Two weights only: `regular` (400) and `semibold` (600). 4 sizes declared.

| Role | Fluent token | Size | Weight | Line Height | Used for |
|------|--------------|------|--------|-------------|----------|
| Body | `fontSizeBase300` | 14px | regular (400) | `lineHeightBase300` έΚΙ 1.43 | Chat message text, input field text, context card value text |
| Label | `fontSizeBase200` | 12px | regular (400) | `lineHeightBase200` έΚΙ 1.33 | Disabled-placeholder hints ("ProviderΎ╝ΙίΞ│ί░Ηί╝ΑόΦ╛Ύ╝Κ"), context card caption, tooltips |
| Heading | `fontSizeBase400` | 16px | semibold (600) | `lineHeightBase400` έΚΙ 1.5 | Empty-state headingήΑΝί╝ΑίπΜϊ╜┐ύΦρ AsterήΑΞ |
| Display | `fontSizeBase500` | 20px | semibold (600) | `lineHeightBase500` έΚΙ 1.5 | Reserved έΑΦ not used in Phase 1 shell; available for Phase 2 onboarding titles |

Rules:
- Body line-height target έΚΙ 1.5 is satisfied by Fluent's paired `lineHeightBase*` tokens έΑΦ do not override.
- Never use `weight="bold"` (700) έΑΦ only `regular` and `semibold` are in the contract.
- All visible strings are wrapped in Lingui macros and ship zh-CN only (FOUND-08 / CONTEXT D-17). Typography applies to the rendered Chinese strings.

**Source:** spike `main.tsx` (already uses `Text size={600} weight="semibold"`), CLAUDE.md (Fluent v9 typography), Fluent v9 token set.

## Color

Driven entirely by Fluent v9 semantic tokens so Office host light / dark / HC themes apply for free. The hex shown is the **light-theme resolved value for reference only** έΑΦ executor references the token.

| Role | Fluent token | Light value (ref) | Usage |
|------|--------------|-------------------|-------|
| Dominant (60%) | `colorNeutralBackground1` | `#ffffff` | Task Pane base surface, chat stream background |
| Secondary (30%) | `colorNeutralBackground2` / `colorNeutralBackground3` | `#fafafa` / `#f5f5f5` | Top context card surface, bottom input-bar surface (the two framing sections that "sandwich" the chat stream) |
| Accent (10%) | `colorBrandBackground` | `#0f6cbd` | Send button primary color slot only (see reserved list) |
| Neutral text | `colorNeutralForeground1` / `colorNeutralForeground3` | `#242424` / `#616161` | Body text / secondary captions + disabled-placeholder hints |
| Destructive | `colorPaletteRedForeground1` | `#b10e1c` | **Not used in Phase 1** έΑΦ no destructive actions exist in the shell (see Copywriting). Reserved for Phase 2 settings/key-delete. |

**Accent reserved for (explicit list):**
1. The bottom **ήΑΝίΠΣώΑΒήΑΞ(Send) button** `appearance="primary"` color slot έΑΦ it keeps the brand color even while disabled in Phase 1 (disabled state dims it via Fluent's own disabled token, but the color slot stays brand).
2. The **context-card refresh "active" indicator** έΑΦ a brief brand-colored pulse/tint on the card when `onSelectionChanged` fires and the value updates, signalling "this is live" (CONTEXT D-12). Subtle, token-driven.

Accent is NOT used for: disabled Provider dropdown, disabled upload icon, empty-state text, context-card text, chat bubbles, borders, or any other interactive element. Everything else is neutral.

**Rationale:** In a shell where nearly every control is a disabled placeholder, restraint on accent is what keeps it reading as "native Office, intentionally not-yet-active" rather than broken. Confirmed by user 2026-05-27.

**Source:** User confirmation 2026-05-27; Fluent v9 semantic token set; CONTEXT D-07/D-08/D-12.
