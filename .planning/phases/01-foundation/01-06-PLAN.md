<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → npm 依赖 | GitHub Actions 安装 npm 依赖并 build（供应链面） |
| GitHub Pages → 公网 | dist/ 静态资源公开可读 |
| 用户 → README sideload 指引 | 用户据 README 下载 manifest 并 sideload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-15 | Tampering | CI npm 依赖 | mitigate | CI 用 `npm ci`（锁 package-lock.json，确定性安装），不用 `npm install`；`permissions: contents: read`（size-guard job 最小权限，无写）。 |
| T-01-16 | Information Disclosure | dist/ Pages 公开 | mitigate | dist/ 构建产物不含任何 API Key/secret（Phase 1 无 Key 逻辑）；README 明确"Key 永不上传 Aster 服务器"，不在仓库/产物嵌入任何凭证。CI 不打印 secret。 |
| T-01-17 | Elevation of Privilege | pages.yml 权限 | mitigate | deploy job 保留必要的 `pages: write` / `id-token: write`，但不授予 `contents: write`；不在 workflow 中执行任意用户输入。 |
| T-01-18 | Tampering | sideload manifest 来源 | accept | v1 仅 sideload + 开源仓库 manifest（无 AppSource）；用户从 GitHub 仓库/Release 获取 manifest，信任边界由 GitHub HTTPS + 仓库所有权保证。Phase 7 REL-02 在 Release 页发布。 |
</threat_model>
