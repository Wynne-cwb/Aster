  <how-to-verify>
  参照 `.claude/skills/office-addin-browser-uat/SKILL.md` 中的真机验证范式。

  **Spike 验证 — 记录每条结果（✅ 可用 / ❌ 不可用 + 错误信息）：**

  **S1 — PPT 取图 spike（最高风险，Priority P0）：**
  1. Office for Web（Edge / Chrome）打开一个 PPT 文件
  2. 点击一张图片 shape 使其选中
  3. 在 Aster Task Pane 中输入「分析这张图」发送
  4. 观察：是否显示「正在看这张图…」工具气泡 + 据图返回文字描述
  5. 若失败：应看到「当前无法读取选中图（宿主限制），可点回形针上传这张图」错误消息

  **S2 — Excel 取图表 spike：**
  1. Excel for Web 打开含图表的 xlsx
  2. **单击图表**使其进入激活状态（重要：不是点单元格）
  3. 输入「解读这个图表的数据」发送
  4. 观察：取图成功 → 据图描述；未激活 → 「请先点击一下图表使其激活」

  **S3 — Word 取 inline picture spike：**
  1. Word for Web 打开含内嵌图片的 docx
  2. 光标点击选中内嵌图片（不是 floating picture）
  3. 输入「这张图片的内容是什么」发送
  4. 观察：取图成功 → 据图描述

  **S4 — Ctrl+V 粘贴 spike：**
  1. 在任一宿主 Task Pane 中，截图（或复制一张图片到剪贴板）
  2. 聚焦 Aster 输入框，Ctrl+V 粘贴
  3. 观察：应看到图片缩略图 chip 出现在输入框上方
  4. 输入「分析这张图」发送，观察是否据图作答

  **端到端 UAT（至少完成以下任意两个场景）：**

  **UAT-1 — 回形针上传图（SUCCESS CRITERIA 2）：**
  1. 点回形针按钮，选择一张 png/jpg/webp 图片（≤5MB）
  2. 观察：缩略图 chip 显示
  3. 输入「基于这张图写一段描述」发送
  4. 观察：agent 据图作答（不是说「我看不到图」）
  5. 不刷新，继续追问「再详细说一下颜色」——观察多轮复用是否正常（D-10）

  **UAT-2 — 非图片文件诚实提示（SUCCESS CRITERIA 3 / D-14）：**
  1. 点回形针按钮，选择一个 .docx 或 .pdf 文件
  2. 观察：应提示「文件解析即将开放，当前可上传图片」（不假装支持）

  **UAT-3 — 没配 aihubmix key 错误（SUCCESS CRITERIA 3 / D-13）：**
  1. Settings 面板删除 aihubmix Key（或用无效 key）
  2. 尝试「看图」操作
  3. 观察：应提示「请先在设置里填写 aihubmix Key」

  **UAT-4 — 选区不是图错误（SUCCESS CRITERIA 3 / D-13）：**
  1. PPT 中选中一个文字 shape（而非图片）
  2. 输入「分析这个形状」发送
  3. 观察：应提示「请先选中一张图片或图表，或点回形针上传一张图」
  </how-to-verify>
