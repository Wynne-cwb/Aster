## Spacing Scale

线上 `src/styles.css :root` 已规整为 `--space-1..6`（4 的倍数，符合 8-point 精神）。本 phase **只用既有 token**，不新增。

| Token | Value | Usage（本 phase） |
|-------|-------|-------|
| `--space-1` | 4px | 图标与文字 gap、徽章内边距、行内微距 |
| `--space-2` | 8px | 卡片内 gap、按钮间距、卡片上边距（`margin-top`） |
| `--space-3` | 12px | 卡片横向 padding、modal footer 纵向 padding |
| `--space-4` | 16px | modal 内 padding、modal footer 两按钮 gap（**UAT 修，必须 16px 不是 8px**） |
| `--space-5` | 20px | modal 容器 padding |
| `--space-6` | 24px | 大区块间距（本 phase 基本用不到） |

折叠行内沿用 writeback 范式的非 scale 内边距：`.wb-action-head` = `6px 10px`、`.wb-action-body` = `6px 10px 8px`、`.tool-group__head` = `8px 12px`、`.tool-group__list > li` 行内去圆角靠 1px border 分隔（**复用，不改**）。

Exceptions: 折叠行 `6px 10px` / 合并卡 head `8px 12px` 沿用 `.tool-group` / `.wb-action-head` 既有实测值（writeback reference line 38-68）；这些不是新约定，是复用既有组件不可拆的内距。

## Color

60/30/10 在本 phase 的落点（全部走既有语义变量，light/dark 各自一套）：

| Role | Value（light / dark） | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` / `--surface` = `#FFFFFF` / `#0E0E10`·`#181819` | pane 底、卡片底、modal 底 |
| Secondary (30%) | `--surface-2` `#f3f2ee`/`#1f1f21`、`--border` `#e7e5df`/`#26262a`、`--text-2`/`--text-3` | 卡片边框、折叠行 hover 底、次要文字（humanLabel 行用 `--text-2`，mono/source 用 `--text-3`） |
| Accent (10%) teal | `--accent` `#009887` / `#4FC9B8` | 见下「Accent reserved for」 |
| Destructive | `--error` `#dc2626` / `#f87171`、`--error-soft` | 仅「未能回滚（宿主报错）」三态之一 + 错误描边；不用于普通撤销按钮 |

**Accent reserved for（teal 仅限以下，绝不泛用到所有交互元素）：**
- 焦点 ring（`--ring-focus` 双环，所有可聚焦控件）。
- undo-all **二次确认 modal 的主按钮**「确认撤销」（`.btn-primary` teal 实底）—— 这是本 phase 唯一的 teal 实底主按钮。
- copy step log **成功反馈态**的 `CheckIcon`（teal currentColor，沿用 InputBar `已复制 ✓` 反馈语言）。
- per-step 状态胶囊「已撤销」（`--accent-soft` 底 + `--accent` 字，沿用 `.badge-accent`）—— 表示「已成功回滚」的中性正向态。

**三态状态色（D-12 undo-all 总结 + per-step 标记，映射既有语义 token）：**

| 状态 | 文案 | 色 token | 视觉 |
|------|------|----------|------|
| 已回滚（成功） | 已撤销 | `--success` / `--success-soft`（绿）或 `--accent-soft`（teal，二选一，见下注） | 胶囊 success-soft 底 + success 字（沿用 `.badge-success`） |
| 未回滚（你已手动修改） | 未回滚 · 手改 | `--warning` / `--warning-soft`（琥珀） | 该步行加 `.is-undone` 中性灰删除线 + warning 胶囊；**非红**（不是错误，是有意保留用户改动） |
| 未能回滚（宿主报错） | 未能回滚 | `--error` / `--error-soft`（红） | 行内 `li.is-error .wb-action-target { color: var(--error) }`（沿用 `.tool-group` 错误行）+ error 胶囊 |

> 注：「已回滚」胶囊用 **`--success`（绿）** 表达「回滚成功」语义最直观，与 undo-all 总结里的 X 计数色一致；teal `--accent-soft` 留给「已撤销」这个 per-step 静态终态标记（撤销态胶囊，writeback reference `.is-undone` 规格）。executor 落地时二者保持 success=绿、撤销终态=teal-soft 的区分即可，避免绿/teal 在同一行打架。

dark 下 `--success / --warning / --info` 若未在 `[data-theme="dark"]` 重定义会继承 light 值 —— **executor 必须 dark 逐项 QA**（design-tokens reference 已标 Known Gap：dark 下绿/琥珀对比度需确认；`--error` dark 已提亮为 `#f87171`，绿/琥珀同理可能需提亮，按需在 dark 块补）。

## Copywriting Contract

全部 zh-CN + Lingui macro。动态量词（N 处 / X 步）用 Lingui plural/占位符，不手拼字符串。

| Element | Copy |
|---------|------|
| DiffLogPanel 汇总卡 head | `本次改动 {N} 处`（N = 该 runId 写操作数；D-01/D-03 只列有 reverse 的写操作。展开 chevron 在左，rotate(180) 展开） |
| 每步行（展开后） | `{humanLabel}`（中文人话，来自各 write tool 的 `humanLabel(args)`，**不显示** raw JSON / tool name 给用户）+ 右侧「撤销该步」按钮 |
| per-step undo 按钮 | `撤销该步`（`.btn-ghost .btn-sm` 灰按钮气质，不抢主流程；D-04） |
| per-step 已撤销态 | 行加 `.is-undone`：humanLabel 删除线 + 中性灰，撤销按钮消失，行尾换上「已撤销」胶囊（`.badge-accent` teal soft）；底部一行细提示 `已撤销，文档已回滚到上一状态`（`--text-3` fs-11） |
| Primary CTA（本 phase 唯一 teal 实底主按钮） | `确认撤销`（undo-all 二次确认 modal 主按钮；危险动作的「执行」键，teal 实底 = 让用户清楚这是要执行的那一下） |
| Undo-all 触发按钮 | `撤销本次所有操作`（卡片底部 `.btn-ghost` 灰描边按钮，**不和主流程主按钮混**；D-04 / AGENT-09。点击 → 弹二次确认 modal，不直接执行） |
| 二次确认 modal 标题 | `撤销本次所有操作？` |
| 二次确认 modal 正文 | `将逆序撤销本次 AI 改动的 {N} 处写操作。你手动改过的内容会自动跳过、保留不动。`（说清「逆序 + 跳过手改」两条用户最关心的行为；D-07/D-10） |
| 二次确认 modal 取消键 | `取消`（`.btn-ghost`，左；右为 teal 实底「确认撤销」，gap 16px UAT 值） |
| Undo-all 总结 modal 标题 | `撤销完成`（D-12；若 X=0 全跳过则标题不变，靠正文区分） |
| Undo-all 总结 modal 正文（三态，按发生才显示对应行） | 行1 成功（绿）：`已回滚 {X} 步` · 行2 手改（琥珀，Y>0 才显示）：`跳过 {Y} 步（你已手动修改，保留不动）` · 行3 报错（红，Z>0 才显示）：`{Z} 步未能回滚（宿主报错）`（D-12 三态色一一对应上表） |
| Undo-all 总结 modal 关闭键 | `知道了`（`.btn-primary` 或 `.btn-ghost`，单按钮右对齐；纯告知态无需取消） |
| Empty state | **无独立空状态**（D-02：汇总卡只在 run 完成且写操作 > 0 时出现；0 写操作 → 不渲染汇总卡，聊天流照常。无「暂无改动」占位） |
| Error state（per-step 撤销自身报错） | 该步标红「未能回滚」+ `errorHint`「宿主 API 报错，无法回滚此步」；undo-all 继续撤剩余步（D-11 最大努力，不遇错即停） |
| copy step log 按钮（主界面） | InputBar 工具行图标按钮，`title`/`aria-label` = `复制操作记录`（与现有「复制调试信息」并列；点击后 2 秒反馈 `已复制 ✓` + CheckIcon teal） |
| copy step log 按钮（Settings 入口） | Settings 全局选项区一行 `复制本次操作记录`（`.btn-ghost .btn-sm` 或行式入口；D-22 双入口） |
| copy 脱敏说明 | 按钮 `title` 或 Settings 行下细字 `已自动隐去 API Key 与 Provider id`（D-21；让用户敢贴 issue） |

**Destructive 动作清单 + 确认方式：**
- 「撤销本次所有操作」= 唯一需二次确认的破坏性动作 → `.modal-scrim` + `.modal` 二次确认（复用 OnboardingModal 的 modal 范式，**不新造 dialog 组件**）。teal 实底「确认撤销」+ ghost「取消」，foot 右对齐 gap 16px。
- 「撤销该步」= 单步可逆、低破坏性 → **不弹确认**，直接执行 + 行内变 `.is-undone`（与 AGENT-09 一致：单步轻、整体重）。
- copy step log = 非破坏性 → 无确认，点击即复制 + 反馈。

## Interaction Contract

| 行为 | 契约 |
|------|------|
| 汇总卡出现时机 | 仅 `agentStatus === 'idle'`（run 完成）且该 runId 写操作 > 0；run 进行中靠现有 live `role='tool'` 卡，**不另起常驻面板/抽屉**（D-01/D-02） |
| 多轮 run | 每个 runId 一张汇总卡，旧卡保留可见、各自只撤自己那轮（D-08）；DiffLogPanel 按 runId 隔离，不订阅全局（防混渲 — RESEARCH Pitfall 3） |
| 折叠交互 | 复用 `wb-action-head`：head 点击 toggle，chevron `rotate(180deg)` `transition: transform var(--dur-fast)`；展开才显示每步行 + 底部 undo-all 按钮 |
| 单步撤销 | 点「撤销该步」→ 直接执行（无确认）→ 成功：行变 `.is-undone`；失败（宿主报错）：行标红「未能回滚」；手改跳过：标 warning「未回滚·手改」 |
| undo-all 流程 | 点「撤销本次所有操作」→ 二次确认 modal →「确认撤销」→ 逆序 replay → 完成弹总结 modal（D-12）。「取消」关 modal 不执行 |
| 焦点管理 | modal 打开焦点入 modal、ESC / scrim 点击关闭（与 OnboardingModal 一致）；折叠 head / 撤销按钮 / modal 按钮焦点用 `--ring-focus`（modal 输入类用更柔的 3px accent-soft，本 phase modal 无输入框，统一 ring-focus） |
| copy 反馈 | 点击 → 写剪贴板（`navigator.clipboard.writeText` 主 + `execCommand` fallback，复用 `debugReport.ts` `copyToClipboard`）→ 成功 2s 「已复制 ✓」→ 复原；失败静默不崩（沿用现有 try/catch 范式） |
| 动效 | 全部 120–320ms（`--dur-fast/base/slow` + `--ease-out`）；`prefers-reduced-motion` 全局已降级，新元素无需单独处理 |
| 诚实禁用 | 本 phase 无「即将开放」占位；若某步已撤销，撤销按钮**移除**而非禁用（避免 disabled 误导可再点） |

## Bundle Budget Note（NFR-05 — CI ≤ 82KB gzip，当前余 ~1.46KB，极紧）

UI 层对 bundle 的影响与建议（给 planner / executor，非视觉契约但影响实现切分）：

- **DiffLogPanel 建议 `dynamic import`**：run 完成后才需要渲染，非首屏热路径。`React.lazy(() => import('./DiffLogPanel'))` + ChatStream 末尾 `<Suspense>` 包裹，把 panel + undo 触发 UI 移出初始 main chunk。CSS 走 `src/styles.css` 单份（gzip 增量极小，不拆 CSS）。
- **copy step log** 沿用 `debugReport.ts` 已有的「dynamic import + 懒加载」范式（`copyStepLog.ts` 纯函数同样懒加载，按钮 onClick 时才 `await import`）——与现有「复制调试信息」一致，0 初始体积。
- **二次确认 / 总结 modal** 复用既有 `.modal-scrim`/`.modal` CSS（已在 bundle 内），不新增 modal 库；JSX 跟随 DiffLogPanel 的 lazy chunk。
- **图标**：撤销/复制/状态用既有 `TrashIcon`/`ClipboardIcon`/`CheckIcon`/`AlertIcon`/`ChevronDownIcon`（已 export，0 新增）；若缺「撤销/回滚」造型，手写 path 套 `base` 常量新增到 `icons.tsx`（单个 path ~几十字节，不引 lucide-react 整包）。
- **守门**：每个 plan commit 后 `npm run build && npm run size`（注意 size 测 dist，动 bundle 前先 build —— 记忆 [[project_bundle_size_guard]]）。
