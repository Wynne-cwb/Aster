## 关键架构决策（本 phase 锁定）

1. **双入口、共享 `src/`**（ARCHITECTURE Q1 选项 b）：`index-wps.html`（root，Vite 第三 input）+ `src/main-wps.tsx`，不引 office.js CDN。
2. **宿主识别**：`Application.ComponentType`（1=文字/word, 2=表格/excel, 3=演示/ppt）→ `createWpsAdapter(type)`。替代 `Office.onReady`+`Office.context.host`。
3. **ribbon 回调必须是全局 classic 函数**：ES module 内的函数是模块作用域，WPS 按名字找不到。故 `OnAddinLoad`/`ShowTaskPane`/`OnGetEnabled` 放 `public/wps/ribbon-wps.js`（classic script，<head> 同步加载），不放 main-wps.tsx。
4. **storage 零改动**：`src/lib/storage.ts` 的 `partitionKey===undefined` 降级路径在 WPS CEF（`typeof Office==='undefined'`）开箱命中（ARCHITECTURE Pattern 3）。
5. **本 phase adapter 全是 stub**：三宿主 `createWpsAdapter` 都路由，但 read/write/insert 一律 `throw UnsupportedOperationError('WPS adapter 尚未实现 — Phase 32(PPT) / WPS-D1(Excel/Word)')`。真实 read/write 在 Phase 32。
6. **主题**：WPS 无 `Office.context.officeTheme`；先按 `prefers-color-scheme` 兜底（`[真机待验]` WPS CEF 是否暴露系统主题）。

## 反模式守门（ARCHITECTURE §Anti-Patterns，代码 review 自查）

- ❌ 不在同一 html 做 Office-vs-WPS 运行时切换 → 物理分两个 html 入口
- ❌ 不在 WPS adapter 模仿 `*.run()`/`load`/`sync` → 同步调用 + `Promise.resolve`
- ❌ 不在 WPS adapter 调 `Office.isSetSupported` → WPS 无 requirement set，`try/catch` 探测或静态 true/false
- ❌ 不用 `wps.PluginStorage` 存 Key（非持久）→ 裸 `localStorage`

## 风险（本 phase 特有）

- **双 context 双挂载**：WPS 加载 addon 入口页（ribbon context）+ ShowTaskPane 开的 taskpane 都是 `index-wps.html` → React 可能在不可见的 ribbon 背景页也 mount 一次。本 phase 镜像探针的单页结构 + 标 `[真机待观察]`；若真机有副作用，再拆 `index-wps.html`（薄 ribbon）/ `taskpane-wps.html`（React）两页。
- **`window.Application` 注入时序**：module script 在 DOMContentLoaded 跑，可能早于 WPS 注入 → main-wps.tsx 加 readiness poll（类比 Office.onReady 的角色）。
- **base 路径**：Vite `base:'/Aster/'` → 产出 html 引用 `/Aster/assets/*`；GitHub Pages 上 taskpane URL = `…/Aster/index-wps.html`。部署具体 URL 标 `[真机待验]`。
