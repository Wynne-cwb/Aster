<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| textarea 用户输入 → usePreferencesStore.setPrefs | 唯一偏好输入点；sanitizePrefs 在 preferences.ts 统一清洗（不在组件层清洗） |
| clearHistory 按钮 → storage.remove(docKey) | 用户主动操作；docKey 来自 getDocKey()（本机 Office 环境），不含外部输入 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-ui-injection | Tampering | textarea → setPrefs | mitigate | sanitizePrefs 在 preferences.ts 执行（单一清洗点）；组件只做 onChange → setPrefs 透传；UI 层不负责 sanitize |
| T-08-clear-scope | Tampering | clearHistory(docKey) | accept | clearHistory 只删 docKey 对应的单个 localStorage key（D-12）；不清其他文档；docKey 来自受控的 getDocKey() |
| T-08-css-inject | Injection | CSS pref-input 样式 | accept | 纯 CSS 类名，无动态样式；无 dangerouslySetInnerHTML；textarea value 由 React 控制不注入 HTML |
</threat_model>
