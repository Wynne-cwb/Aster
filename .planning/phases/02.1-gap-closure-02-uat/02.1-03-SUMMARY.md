# Phase 02.1 Plan 03: ChatStream 粘底状态机修复 (G-03) Summary

**One-liner:** 引入 stickToBottom 状态机（含 onScroll 阈值检测、prevLengthRef 新消息强制滚动），把 useEffect 依赖从 messages.length 改为 messages 整体引用，修复流式 delta 期间不自动滚到底的问题（G-03）。

## What Was Built

修复了 ChatStream.tsx 中流式输出不自动滚到底的根因（D-10 印证）：

**根因：** 原 useEffect 依赖 `[messages.length]`，流式 token 追加只改 `messages[i].content`，length 不变 → useEffect 不触发 → `scrollTo` 不调用。

**修复：**

1. **useEffect 依赖改为 `[messages]`**：chatStore 每次 set 返回新数组引用，流式 delta 也触发。
2. **stickToBottom 状态机**：初始 true；onScroll 检测 `scrollTop + clientHeight >= scrollHeight - 8` 更新；用户上滑 → false（不打断）；用户滚回 → true（恢复粘底）。
3. **prevLengthRef 新消息检测**：length 增加时强制 smooth 滚动（不论 stickToBottom），否则仅 stickToBottom 时 auto 滚动。
4. **vitest 测试**：4 个用例覆盖三种粘底分支，使用 @testing-library/react + vi.spyOn + jsdom HTMLElement.prototype.scrollTo 空实现。
