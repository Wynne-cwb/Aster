按 D-06 三步排查清单修订 `src/styles.css`（最小改动，沿用现有 token，禁止引入新颜色/尺寸变量）：

① **`.aster-shell`**（styles.css:108）— 已有 `overflow-x: hidden`，**额外**加 `min-width: 0`，避免被 flex 父级（理论上 Task Pane 是 Office iframe，但保险起见）撑破。

② **`.aster-chat`**（styles.css:198）— 加 `min-width: 0`（flex 子项默认 `min-width: auto`，长内容会撑破 flex 容器，这是经典 trap）。

③ **`.aster-messages`**（styles.css:1134）— 加 `min-width: 0` + `overflow-wrap: anywhere`，让 `<pre>`/`<code>`/长 URL 在容器内换行而不撑破。

④ **`.aster-inputbar`**（styles.css:269）— 加 `min-width: 0`。

⑤ **`.aster-inputbar__pill-row`**（styles.css:1144）— 加 `min-width: 0` + `overflow: hidden`（pill 长度通过自身 max-width:200px 控制，但外层行也要兜底）。

⑥ **`.aster-bubble`**（styles.css:390）— 加 `overflow-wrap: anywhere` + `word-break: break-word`。

⑦ **`.aster-bubble--assistant pre`**（styles.css:422）— 已有 `overflow-x:auto`；**额外**加 `max-width: 100%` + `white-space: pre-wrap`（让代码块超长行在窄面板下换行，不再要求横向滚动整块代码——窄面板里横向滚代码体验更差）。说明：`pre-wrap` 会丢失原始的「不换行」语义，但 350px 面板这是必要折中，CLAUDE.md §UI 设计系统的「美观优先」要求在窄面板里保持垂直流式。

⑧ **`.aster-bubble--assistant code`**（styles.css:430）— inline code 加 `overflow-wrap: anywhere`。

⑨ **`.aster-bubble--assistant a`**（新建选择器，react-markdown 渲染的链接）— 加 `overflow-wrap: anywhere` + `word-break: break-all`，防止单一长 URL 撑破气泡。

不修改 ContextCard、SelectionPill 本身（它们已有 `overflow:hidden` + `text-overflow:ellipsis`）。不引入 `@media` query（350px 是 Task Pane 默认下限，无需响应式分支）。
  </action>
