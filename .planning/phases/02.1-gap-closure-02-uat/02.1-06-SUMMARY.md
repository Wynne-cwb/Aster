# Phase 02.1 Plan 06: Settings ProviderForm sticky footer + 三分区路由修复 Summary

**One-liner:** ProviderForm 三层 sticky footer（`__header/__body/__footer`）+ SettingsPanel `EditState` 状态机，编辑态独占 body、全局选项只在浏览态渲染，彻底消除 G-06 「按钮在中部、下方跟全局控件」的布局混乱。

## What Was Built

### Task 1: ProviderForm sticky footer

`ProviderForm.tsx` 从单层 form 拆为三层 BEM 结构：
- `.aster-provider-form__header`：标题（`flex-shrink:0`，不随内容滚动）
- `.aster-provider-form__body`：所有字段 + 隐私 hint（`flex:1 overflow-y:auto`，字段多时可滚动）
- `.aster-provider-form__footer`：「保存 / 取消」按钮（`position:sticky bottom:0`，始终贴底）

`styles.css` 修改：
- `.aster-provider-form` 改为 `display:flex flex-direction:column height:100% min-height:0`（flex column 容器）
- 新增 `__header/__body/__footer` 三层样式
- 删除废弃的 `.aster-form-actions`（已被 `__footer` 替代）

### Task 2: SettingsPanel 三分区路由

`SettingsPanel.tsx` 引入 `EditState` 状态机：
```ts
type EditState =
  | { kind: 'browse' }
  | { kind: 'editing'; providerId: string }
  | { kind: 'creating' };
```

路由逻辑（D-26 实现）：
- `browse` 态：渲染 `<ProviderList>` + `.aster-settings__global-options`（自动附带选区开关 + 重看引导）
- `editing/creating` 态：仅渲染 `<ProviderForm>`，列表与全局选项完全不渲染

`ProviderList.tsx` 改造（状态上抛）：
- 移除内部 `editingProvider` state 和嵌套的 `<ProviderForm>` 渲染
- 新增 `onEdit(providerId: string)` 和 `onCreate()` props
- 深链 `focusAnchor` 改为 useEffect 触发 `onEdit(defaultProviderId)`，由 SettingsPanel 持有编辑态
- 删除相关 `addProvider/updateProvider/setKey` 调用（全部上移到 SettingsPanel 的 `handleSave`）

`styles.css` 新增 `.aster-settings__global-options` 容器（全局选项分区）。

### Minor adjustments

**1. [Rule 1 - Adjustment] `__body` 去掉 `gap`，依赖现有 `aster-form-field` margin**
- **Found during:** Task 1 CSS 编写时
- **Issue:** PLAN.md 示例在 `__body` 设了 `gap: var(--sp-3)`，但 `.aster-form-field` 已有 `margin-bottom: var(--sp-3)`，两者叠加会产生双倍间距
- **Fix:** 移除 `__body` 的 `gap` 属性，让字段间距完全由 `.aster-form-field` 的 `margin-bottom` 控制
- **Files modified:** src/styles.css

**2. [Rule 1 - Adjustment] `aster-settings__global-options` 样式保守化**
- **Issue:** PLAN.md 示例中 `global-options` 有 `margin-top / padding-top / border-top`，但现有 `.aster-settings__section` 已有 `border-bottom: 1px solid var(--border)` 且 `padding` 充足，额外分隔线会导致双重边框
- **Fix:** `global-options` 设为纯 flex 容器（`display:flex flex-direction:column`），不加额外 margin/padding/border，依赖内部 `__section` 的现有样式
- **Files modified:** src/styles.css
