## What Was Built

### Task 1: SSE tool-call 解析 + Provider 探测（D-17 / D-18）

**src/lib/sse.ts：**
- 新增 `ToolCallDelta` / `ToolCallEnd` 类型，`SSEEvent` 联合扩展
- `streamSSE` generator 函数体内声明 `accum Map`（I-10）
- 解析 `choices[0].delta.tool_calls`，按 index 累积 id/name/arguments
- `finish_reason='tool_calls'` 或 `[DONE]` 时 flush accum → yield `ToolCallEnd`
- `sanitizeErrBody` 函数（I-09）：递归脱敏 sk- 值 + apiKey/authorization 字段名
- `!resp.ok` 路径：sanitize errBody 后挂到 AsterError（供 openai-compat 探测用）

**src/providers/openai-compat.ts：**
- `INSERT_TO_DOCUMENT_TOOL` 常量（D-17 完整 schema）
- `_startStream`：查 `providerStore.supportsToolCall !== false` → 条件附带 tools
- `streamChat` catch 块：检测 tool 关键词 4xx → `setSupportsToolCall(id, false)`

**src/providers/types.ts：**
- `ProviderConfig.supportsToolCall?: boolean | null`（null=未探测）

**src/store/providers.ts：**
- `setSupportsToolCall(providerId, supports)` action + localStorage 持久化

**测试（sse.test.ts 新增 5 个）：**
- Test 1: 正常 tool_calls SSE → yield ToolCallEnd 含 arguments
- Test 2: 普通 content delta 不 yield ToolCallEnd
- Test 3: errBody sanitize（sk- 值 + apiKey/authorization 字段脱敏）
- Test 4: supportsToolCall=false → 请求体不含 tools
- Test 7: 并发两个 generator accum 不串污染（I-10 回归）

### Task 2: Settings autoInsertMode + storage（D-19）

- `STORAGE_KEYS.AUTO_INSERT_MODE = 'aster:autoInsertMode'`
- `ProviderState.autoInsertMode: AutoInsertMode` + `setAutoInsertMode` action
- `hydrateFromStorage` 恢复 autoInsertMode
- `SettingsPanel` 全局选项分区新增 segmented control（用户确认/AI 自动）
- `.aster-segmented` + `.aster-segmented__item` + `.is-active` CSS
- storage.test.ts 计数从 6 → 7（Rule 1 bug fix）

### Task 3: chatStore ToolCall 数据流（D-20）

- `ToolCall` 接口：id/name/arguments/status(pending|accepted|rejected)
- `Message.toolCalls?: ToolCall[]`
- `sendMessage` for-await 新增 `tool_call_end` 分支：解析 JSON → 校验 position enum → 按 autoInsertMode 设 status
- `acceptToolCall(messageId, toolCallId, adapter)` — 调 adapter.insert，成功后 status='accepted'
- `rejectToolCall(messageId, toolCallId)` — status='rejected'
- chatStore 不持有 adapter，通过参数注入（符合 D-20）

### Task 4: confirm/auto UI + adapter position + fallback（合并 I-03）

**ChatBubble.tsx（完全重写）：**
- 默认「插入到文档」按钮移除（D-16 修订）
- `ToolCallPreviewCard`：pending → 完整预览卡（200 字截断 + position 中文标签 + 插入/拒绝按钮），accepted → 「已写入 N 字到{位置}」回执，rejected → 「已拒绝写入」
- `AutoInsertEffect`：auto 模式下 status='accepted' 时触发 adapter.insert（副作用分离）
- `FallbackInsertMenu`：仅 !supportsToolCall && !toolCalls 时出现；三位置弹出菜单

**adapters（三宿主 position 路由）：**
- `DocumentAdapter.ts`：`TextContent.position?` 字段（向后兼容，缺省 cursor）
- `PptAdapter.ts`：switch cursor/replace_selection/append_end（PPT 无真光标，cursor=覆盖）
- `ExcelAdapter.ts`：cursor/replace_selection → getSelectedRange，append_end → getUsedRange + A{n+1}
- `WordAdapter.ts`：cursor → after，replace_selection → replace，append_end → body.end

**styles.css：**
- `.aster-tool-card` 系列（pending/accepted/rejected 三态）
- `.aster-insert-menu` + `.aster-bubble__actions`
