# Phase 02.1 Plan 07: G-07 fetch-throw Error Classification Summary

**一句话：** 新增 `classifyFetchThrow` 辅助函数，用 D-29 三条信号将 `fetch` 抛出的 `TypeError` 分类为 `KeyInvalidError`（`sk-invalid` Key 场景）或 `NetworkError`（真断网场景），消除「Key 错误却显示网络失败」的误导。

## What Was Built

- **`src/lib/sse.ts`**: 新增导出函数 `classifyFetchThrow(err, url)` — 实现 D-29 三条信号逻辑（TypeError 消息模式 + navigator.onLine + https 协议），全满足 → `KeyInvalidError('Key 可能无效，请前往设置更新 Key')`，任一不足 → `NetworkError('网络连接失败，请检查网络')`；`streamSSE` 的 fetch 调用改包 try/catch，AbortError 原样上抛，其余走 `classifyFetchThrow`。
- **`src/providers/openai-compat.ts`**: catch 块从 duck typing `'code' in e` 升级为 `instanceof AsterError`，确保 `KeyInvalidError` 等所有 AsterError 子类原样上抛，不被二次包裹成 NetworkError。
- **`src/lib/sse.test.ts`**: 新增 6 个 vitest 用例（describe 块「streamSSE — fetch throw 路径分类（G-07）」）：Test 1-4 覆盖新分类逻辑，Test 5-6 回归验证 mapHttpError 路径未被破坏。

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] AbortError 检测兼容性修复**

- **Found during:** Task 2 GREEN — 运行 sse.test.ts 时 AbortError 用例失败
- **Issue:** `if (err instanceof Error && err.name === 'AbortError')` — 在 jsdom 测试环境中 `DOMException instanceof Error` 为 false，导致 AbortError 绕过保护走到 `classifyFetchThrow`，被错误分类为 NetworkError
- **Fix:** 改为 `if (err != null && (err as { name?: string }).name === 'AbortError') throw err;` — 用 `.name` 属性检查，不依赖 instanceof，兼容 jsdom 和真机浏览器
- **Files modified:** src/lib/sse.ts
- **Commit:** a969e08（同 Task 2）
