<objective>
修复 G-04：DeepSeek（内置）成本徽章只显「N token」，缺「¥X.XXXX」。

根因（按 D-13 三段排查，全部命中其一）：
- **① isBuiltIn 判断错误**：当前没有显式 `isBuiltIn` 判断，但 `calcCostCny(usage, providerId)`（pricing.ts:43-55）仅以 `providerId` 查 `PROVIDER_PRICING`。**`PROVIDER_PRICING` 的 key 是 model 名（`deepseek-v4-flash` / `deepseek-v4-pro`），不是 providerId（`deepseek`）**——`PROVIDER_PRICING['deepseek']` = undefined → return null → 徽章无价格。**这是核心 bug**。
- **② chatStore 收 usage 是否正常**：检查 src/store/chat.ts:140-155 现有逻辑，已经在 `event.type === 'usage'` 分支调 `calcCostCny({promptTokens, completionTokens}, config.providerId)` —— 收 usage 这步是对的，但传错了参数。
- **③ baseURL 不能用作内置判断**：D-13 明确不可用 baseURL，必须用 providerId === 'deepseek'。

修复方案（D-13 / D-14 / D-15）：
1. 改 `calcCostCny` 签名为 `(usage, providerId, model)`：
   - `if (providerId === 'deepseek')` → 按 model（如 `deepseek-v4-flash`）查 PROVIDER_PRICING
   - `else if (providerId === 'aihubmix')` → 阶段二无 LLM 单价表（aihubmix 主要 vision/image），仍 return null（与 D-14 一致）
   - `else` → 自定义 Provider，return null（D-17 维持）
2. chatStore.sendMessage 收 usage chunk 时改为 `calcCostCny({...}, config.providerId, config.model)`。
3. 扩充 pricing.test.ts 覆盖四个分支：deepseek+v4-flash / deepseek+v4-pro / deepseek+unknown-model / custom providerId。

Purpose: SC2「成本徽章」在真机 UAT 通过——用 DeepSeek 发一条 ≥ 100 token 的回答，徽章必须显「本次：N token · 约 ¥X.XXXX」非 0。
Output: 修订 pricing.ts、扩充 pricing.test.ts、修订 chat.ts；真机 UAT 截图证据。

回归参照：REQ COST-01（解析 OpenAI-compatible usage 字段）+ COST-02（内置 DeepSeek 显单价、自定义只显 token）。

**A. 修订 `src/providers/pricing.ts`：**

```ts
// 维持 PROVIDER_PRICING 的 key 是 model 名（已正确）
const PROVIDER_PRICING: Record<string, { input: number; output: number } | undefined> = {
  'deepseek-v4-flash': { input: 0.14, output: 0.28 },
  'deepseek-v4-pro':   { input: 1.74, output: 3.48 },
};

export const CNY_PER_USD = 7.25;

/**
 * 计算本次请求的人民币成本。
 *
 * G-04 修订：双键查表（providerId → 内置 namespace，model → 具体单价）。
 * - providerId === 'deepseek'：按 model 查 PROVIDER_PRICING；model 不在表内 → null。
 * - providerId === 'aihubmix'：Phase 2 暂无 LLM 单价表（aihubmix 主走 vision/image），仍 return null（与 D-14 一致）
 * - 其余（用户自定义 Provider）：→ null（D-17 维持）。
 *
 * **不**用 baseURL 判内置（D-13）；用户改 baseURL 时 providerId 不变。
 *
 * @param usage      token 用量
 * @param providerId Provider ID（'deepseek' / 'aihubmix' / 自定义 uuid）
 * @param model      模型名称（'deepseek-v4-flash' / 'deepseek-v4-pro' / 用户自定义 model）
 */
export function calcCostCny(
  usage: { promptTokens: number; completionTokens: number },
  providerId: string,
  model: string,
): number | null {
  // D-13 ①: 内置 isBuiltIn 判断下沉至此（providerId === 'deepseek' 才入价格计算路径，
  //         自定义 Provider 直接 return null）。CostBadge.tsx 不再判 isBuiltIn——本组件
  //         只关心传入的 cnyAmount 是否为 null（null = 不显示 ¥）。
  // 仅 deepseek 内置 namespace 计价（D-14）
  if (providerId !== 'deepseek') return null;

  const pricing = PROVIDER_PRICING[model];
  if (!pricing) return null;  // 即便是 'deepseek' providerId，model 未知也不计价（防御）

  const usd =
    (usage.promptTokens / 1_000_000) * pricing.input +
    (usage.completionTokens / 1_000_000) * pricing.output;

  return usd * CNY_PER_USD;
}
```
