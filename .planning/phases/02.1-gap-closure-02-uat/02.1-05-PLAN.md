② 在 streamSSE 主循环内（chunk JSON 解析后，delta.content 检查之后）追加 tool_calls 处理：

**⚠ I-10 关键作用域要求：** `const accum = new Map(...)` **必须**声明在 `export async function* streamSSE(...)` **生成器函数体内**（每次 invoke streamSSE 都 fresh 一个 Map）。**绝不能**写在模块顶层——否则多个并发请求会共用同一个 Map，tool_call_id 'A' 与 'B' 互相污染。具体放置位置：在 fetch / 进入 reader 循环**之前**，与现有的 SSE buffer 同级声明。

```ts
export async function* streamSSE(
  url: string,
  body: Record<string, unknown>,
  signal: AbortSignal,
): AsyncGenerator<SSEEvent> {
  // ⚠ I-10：accum 必须在 generator 函数体内声明（每次 invoke 新建 Map），
  // 不能写在模块顶层——并发请求共用同一个 Map 会导致 tool_call id 串污染
  const accum = new Map<number, { id: string; name: string; arguments: string }>();

  // ... 原有 fetch + try/catch 块（classifyFetchThrow） ...
  // ... resp.ok 检查 + errBody 挂载（见 D 段 errBody sanitize） ...

  // 在 try { const chunk = JSON.parse(data) ... } 块内：
const toolCalls = chunk.choices?.[0]?.delta?.tool_calls;
if (Array.isArray(toolCalls)) {
  for (const tc of toolCalls) {
    const idx = tc.index ?? 0;
    const prev = accum.get(idx) ?? { id: '', name: '', arguments: '' };
    if (tc.id) prev.id = tc.id;
    if (tc.function?.name) prev.name = tc.function.name;
    if (tc.function?.arguments) prev.arguments += tc.function.arguments;
    accum.set(idx, prev);

    yield {
      type: 'tool_call_delta',
      index: idx,
      id: tc.id,
      name: tc.function?.name,
      argumentsChunk: tc.function?.arguments ?? '',
    };
  }
}

// finish_reason='tool_calls' 时（在 chunk.choices?.[0]?.finish_reason 检查处）：
if (chunk.choices?.[0]?.finish_reason === 'tool_calls') {
  for (const [idx, acc] of accum.entries()) {
    if (acc.id && acc.name) {
      yield { type: 'tool_call_end', index: idx, id: acc.id, name: acc.name, arguments: acc.arguments };
    }
  }
  accum.clear();
}
// 注意：[DONE] 前若 finish_reason='tool_calls' 未来到（少数 provider 不发 finish_reason），
// 在 generator 结束的 return 之前也做一次 flush。
```

为了让 errBody 能拿到，**在 sse.ts 的 mapHttpError 抛错之前**，把 errBody 挂到 error 实例上。**⚠ I-09：errBody 在挂载之前必须先 sanitize，剥除任何看起来像 apiKey 的字段**——否则恶意/有 bug 的 Provider 把请求里的 apiKey 回吐到 errBody，会泄漏到 chatStore.errorMessage 与日志：

```ts
/** I-09：errBody sanitize — 剥除任何看起来像 apiKey 的字段/值后再挂到 AsterError */
function sanitizeErrBody(body: unknown): unknown {
  if (!body || typeof body !== 'object') return body;
  // 递归剥除任何看起来像 apiKey 的字段或值
  return JSON.parse(JSON.stringify(body, (key, val) => {
    if (typeof val === 'string' && /sk-[a-zA-Z0-9]/.test(val)) return '[REDACTED]';
    if (/^(api[_-]?key|authorization|bearer)$/i.test(key)) return '[REDACTED]';
    return val;
  }));
}

if (!resp.ok) {
  const errBody = await resp.json().catch(() => ({}));
  const err = mapHttpError(resp.status, errBody, ...);
  (err as any).errBody = sanitizeErrBody(errBody);   // <- I-09：sanitize 后再挂
  (err as any).httpStatus = resp.status;
  throw err;
}
```
