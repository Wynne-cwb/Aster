<objective>
修复 G-03：ChatStream 在 AI 流式逐字输出时不自动滚到底，用户需手动滚动（真机截图：长回答中段，下半部分被裁，需用户滚）。

根因（D-10 印证）：
- 现 ChatStream.tsx:55-57 的 useEffect 依赖是 `[messages.length]`。
- 流式期间 token 追加只改某条 `messages[i].content`，**`messages.length` 不变 → useEffect 不触发 → scrollTo 不调用**。

修复策略（D-11 + D-12）：
1. 把 useEffect 依赖改为 `[messages]`（整体引用变化）—— chatStore 每次 set 都返回新数组（已经是 immutable 风格），引用每次都变。
2. 引入「粘底」状态机 `stickToBottom`：
   - 初始 `true`（首次渲染就粘底）
   - 用户向上滚动（`scrollTop + clientHeight < scrollHeight - 8`）时，set `false`（不再粘底，让用户安心读历史）
   - 用户滚回底部（`scrollTop + clientHeight >= scrollHeight - 8`）时，set `true`（恢复粘底）
3. useEffect 触发时，只有 `stickToBottom === true` 才 `scrollTo(scrollHeight)`。
4. 新增消息（messages.length 增加，identified by 上一次 length 缓存）始终强制粘底（用户发新消息或 AI 起新回复，预期视口跟上）。

Purpose: SC2「流式渲染 + 停止」在真机 UAT 通过——长回答自动滚到底；中途上滑读历史不被打断；滚回底部重新粘底。

```tsx
import { useEffect, useRef, useState } from 'react';
// ... 其余导入保持不变 ...

export default function ChatStream({ onSettings }: ChatStreamProps): ReactElement {
  const host = useAdapter().capabilities().host;
  const examples = usageExamples(host);
  const logo = `${import.meta.env.BASE_URL}assets/icon-80.png`;

  const messages = useMessages();
  const retryMessage = useChatStore((s) => s.retryMessage);

  // G-03 粘底状态机：默认 true；用户上滑 → false；滚回底部 → true
  const scrollRef = useRef<HTMLDivElement>(null);
  const [stickToBottom, setStickToBottom] = useState(true);
  const prevLengthRef = useRef(messages.length);

  // 检测是否「在底部」(8px 阈值，避免亚像素抖动)
  const isAtBottom = (el: HTMLElement): boolean => {
    return el.scrollTop + el.clientHeight >= el.scrollHeight - 8;
  };

  // onScroll：用户主动滚动时更新 stickToBottom
  const handleScroll = (): void => {
    const el = scrollRef.current;
    if (!el) return;
    setStickToBottom(isAtBottom(el));
  };

  // 流式追加 / 新消息时滚到底（条件：新消息强制；否则仅 stickToBottom 时）
  useEffect(() => {
    const el = scrollRef.current;
    if (!el) return;

    const isNewMessage = messages.length > prevLengthRef.current;
    prevLengthRef.current = messages.length;

    if (isNewMessage || stickToBottom) {
      el.scrollTo({ top: el.scrollHeight, behavior: isNewMessage ? 'smooth' : 'auto' });
    }
  }, [messages, stickToBottom]); // 依赖 messages 整体引用（流式 delta 也触发）

  // ... 空态分支保持不变 ...

  // 有消息：渲染时绑定 onScroll
  return (
    <div className="aster-messages" ref={scrollRef} onScroll={handleScroll}>
      {messages.map((m) => (
        <ChatBubble
          key={m.id}
          message={m}
          onRetry={() => void retryMessage(m.id)}
          onSettings={onSettings}
        />
      ))}
    </div>
  );
}
```

关键设计点：
1. `stickToBottom` 默认 true（首次渲染就粘底）。
2. 用 `prevLengthRef` 缓存上次 messages.length，判断「新消息触发」与「流式追加触发」的区别。
3. 新消息 → smooth 滚动（视觉舒服）；流式 delta → auto（瞬时，跟随 token 速度）。
4. onScroll 持续更新 stickToBottom——用户上滑 5px 就退出粘底，回到 8px 内重新粘底。
5. **不要**在 useEffect 里调用 setStickToBottom（会引发循环 effect）。
