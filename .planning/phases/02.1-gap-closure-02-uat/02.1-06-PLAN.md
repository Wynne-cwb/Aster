修复 G-06：Settings 编辑 Provider 表单的「保存 / 取消」按钮在表单中部 inline 出现，下方还跟着「自动附带选区」开关 + 「重看引导」链接。真机截图显示编辑态下结构混乱。

根因（D-25 / D-26）：
- ProviderForm.tsx 当前 form-actions 是普通流式 `<div class="aster-form-actions">`（line 187-198），无 sticky footer。
- SettingsPanel.tsx 把 `<ProviderList>` 和「自动附带选区」开关 + 「重看引导」按钮全放在同一 `.aster-settings__body` 流式垂直流里——当 ProviderList 进入编辑态时，ProviderForm inline render 在 ProviderList 内部，**下方还跟着全局选项**，违反 D-26「全局选项绝不能与当前编辑表单混排」。

修复策略（D-25 / D-26）：
1. **ProviderForm 改 sticky footer**：
   - 拆 `<form>` 为「头部（标题）+ body（字段滚动）+ footer（按钮 sticky）」三层
   - footer 用 `position: sticky; bottom: 0;` + 背景色（与 settings 主体一致避免穿透）+ 上边阴影分隔（CLAUDE.md §UI 设计系统已有 --shadow-glass / --shadow-card token）
2. **SettingsPanel 三分区路由**：
   - 引入 `editingProviderId: string | null` state（也可下沉到 ProviderList 自己控制 + onEdit/onCreate 上抛）。**优先方案**：把状态提升到 SettingsPanel —— 编辑时**整页**只渲染 ProviderForm，列表+全局选项**完全隐藏**，避免互相干扰
   - 浏览状态（editingProviderId=null）：渲染 `<ProviderList />` + 全局选项分区
   - 编辑状态（editingProviderId!=null）：仅渲染 `<ProviderForm provider={...} onSave={...} onCancel={() => setEditingProviderId(null)} />`，列表与全局选项都不渲染
3. **「自动附带选区」开关、「重看引导」按钮** 在 SettingsPanel 浏览态下保留为「全局选项」分区（D-26 ③），不参与编辑态。

修订 `src/components/Settings/SettingsPanel.tsx`：

```tsx
import { useState } from 'react';
import { Trans, useLingui } from '@lingui/react/macro';
import { useProviderStore } from '../../store/providers';
import { ChevronIcon } from '../icons';
import ProviderList from './ProviderList';
import ProviderForm, { ProviderFormData } from './ProviderForm';
import type { ProviderConfig } from '../../providers/types';

interface SettingsPanelProps {
  onClose: () => void;
  initialAnchor?: string;
  onShowOnboarding?: () => void;
}

type EditState =
  | { kind: 'browse' }
  | { kind: 'editing'; providerId: string }
  | { kind: 'creating' };

export default function SettingsPanel({ onClose, initialAnchor, onShowOnboarding }: SettingsPanelProps) {
  const { t } = useLingui();
  const autoAttach = useProviderStore((s) => s.autoAttach);
  const setAutoAttach = useProviderStore((s) => s.setAutoAttach);
  const providers = useProviderStore((s) => s.providers);
  const addProvider = useProviderStore((s) => s.addProvider);
  const updateProvider = useProviderStore((s) => s.updateProvider);
  const setKey = useProviderStore((s) => s.setKey);

  // 编辑/新建状态
  const [editState, setEditState] = useState<EditState>(
    initialAnchor === 'key-input' || initialAnchor === 'model-input'
      ? { kind: 'editing', providerId: 'deepseek' }   // 深链默认编辑 deepseek
      : { kind: 'browse' },
  );

  // 编辑态 - 当前 Provider
  const editingProvider: ProviderConfig | undefined =
    editState.kind === 'editing' ? providers.find((p) => p.id === editState.providerId) : undefined;

  const handleSave = (data: ProviderFormData): void => {
    if (editState.kind === 'editing') {
      updateProvider(editState.providerId, { name: data.name, baseURL: data.baseURL, model: data.model });
      if (data.apiKey) setKey(editState.providerId, data.apiKey);
    } else if (editState.kind === 'creating') {
      addProvider({ name: data.name, baseURL: data.baseURL, model: data.model, isBuiltIn: false });
      // 注意：新建后需要找回该 provider 的 id 写 Key —— 由于 addProvider 生成 uuid，store 内部已知；
      // 实务上 setKey 在 addProvider 后用最新 providers 列表的最后一个 id（或扩展 addProvider 返回 id）
      // 此处为简化，若新建后没有立即写 Key 也允许（用户后续编辑写入）
    }
    setEditState({ kind: 'browse' });
  };

  const handleCancel = (): void => setEditState({ kind: 'browse' });

  return (
    <div className="aster-settings">
      <div className="aster-settings__header">
        <button className="aster-iconbtn" onClick={onClose} aria-label={t`返回`} title={t`返回`}>
          <ChevronIcon />
        </button>
        <span className="aster-settings__title"><Trans>设置</Trans></span>
      </div>

      <div className="aster-settings__body">
        {/*
          D-26 SettingsPanel 分区：① 当前编辑表单 / ② Provider 列表 / ③ 全局选项
          实现策略：
            - 浏览态 (editState.kind === 'browse')：仅渲染 ②③（① 隐藏，因为没东西可编辑）
            - 编辑态 (editState.kind === 'editing' | 'creating')：仅渲染 ①（独占整个 body，sticky footer 保存/取消）
          这是 D-26 字面顺序的 UX 优化实现——同时显示 ①②③ 在 350px 窄面板里太拥挤，
          且 D-26「全局选项绝不能与当前编辑表单混排」语义已通过「编辑态独占」更强地兑现。
        */}
        {editState.kind === 'browse' ? (
          <>
            {/* ② Provider 列表 */}
            <ProviderList
              focusAnchor={initialAnchor}
              onEdit={(id) => setEditState({ kind: 'editing', providerId: id })}
              onCreate={() => setEditState({ kind: 'creating' })}
            />

            {/* ③ 全局选项分区（D-26 ③） */}
            <div className="aster-settings__global-options">
              <div className="aster-settings__section">
                <label className="aster-settings__toggle-row" htmlFor="setting-auto-attach">
                  <span className="aster-settings__label"><Trans>自动附带选区内容</Trans></span>
                  <input
                    id="setting-auto-attach"
                    type="checkbox"
                    className="aster-toggle"
                    checked={autoAttach}
                    onChange={(e) => setAutoAttach(e.target.checked)}
                    aria-label={t`自动附带选区内容`}
                  />
                </label>
                <p className="aster-settings__hint">
                  <Trans>发送消息时自动附带您当前选中的文档内容</Trans>
                </p>
              </div>

              {onShowOnboarding && (
                <div className="aster-settings__section">
                  <button className="aster-link-btn" onClick={onShowOnboarding}>
                    <Trans>重看引导</Trans>
                  </button>
                </div>
              )}
            </div>
          </>
        ) : (
          /* ① 当前编辑表单（独占整个 body 区域，列表/全局选项不渲染）*/
          <ProviderForm
            provider={editingProvider}
            initialFocus={initialAnchor}
            onSave={handleSave}
            onCancel={handleCancel}
          />
        )}
      </div>
    </div>
  );
}
```
