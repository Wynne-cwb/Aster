# Phase 02.1 Plan 04: G-04 DeepSeek 成本徽章 ¥ 价格修复 Summary

**One-liner:** 修复 `calcCostCny` 双键查表 bug——由 `(usage, providerId)` 改为 `(usage, providerId, model)`，消除 DeepSeek 成本徽章永远返回 `null` 的根因。

## What Was Built

### 根因

`PROVIDER_PRICING` 的 key 是 model 名（`deepseek-v4-flash` / `deepseek-v4-pro`），但调用方传的是 `providerId='deepseek'`，导致 `PROVIDER_PRICING['deepseek']` = undefined → return null → 徽章无价格。

### 修复内容

**1. `src/providers/pricing.ts` — 签名重写**
- 函数签名：`calcCostCny(usage, providerId, model)` → number | null
- 路由逻辑：`providerId !== 'deepseek'` → null（D-17 维持）；`PROVIDER_PRICING[model]` 二级查表
- 加 D-13 ① 锚点注释：isBuiltIn 判断下沉至此

**2. `src/store/chat.ts` — 调用方同步**
- `calcCostCny({...}, config.providerId, config.model)` 传第三个参数 `config.model`

**3. `src/providers/pricing.test.ts` — 双层升级**
- 原 8 个旧 2-参数用例升级为 3-参数（`'deepseek', 'deepseek-v4-flash'` 形式）
- 新增 5 个 G-04 专项用例（Test A-E）覆盖 D-13 三段排查

**4. `src/components/CostBadge.tsx` — D-13 ① 锚点注释（无运行时逻辑变化）**
