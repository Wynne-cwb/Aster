按 D-08「把 .aster-inputbar__pill-row 与 InputBar 主体的 horizontal padding 统一到同一个 CSS 变量」执行：

① 在 styles.css 的 `:root` token 段（约 line 10-37 现有 token 列表之后）新增：
```css
--inputbar-padding-x: var(--sp-4);   /* 16px — InputBar 水平内缩统一基准 */
```
两套主题块（light/dark）**不需要**额外覆盖（值与主题无关）。

② 修订 `.aster-inputbar`（styles.css:269） padding 写法：
   - 原：`padding: var(--sp-3) var(--sp-4);`
   - 改：`padding: var(--sp-3) var(--inputbar-padding-x);`

③ 修订 `.aster-inputbar__pill-row`（styles.css:1144）：
   - 原：`padding: 0 var(--sp-4) var(--sp-2);`
   - 改：`padding: 0 0 var(--sp-2);`（**外层 .aster-inputbar 已经给了 horizontal padding，pill-row 自己不再加横向 padding，避免双倍内缩**）

④ 修订 `.aster-composer`（styles.css:278） padding：
   - 原：`padding: var(--sp-2) var(--sp-2) var(--sp-2) var(--sp-3);`（左 12 右 8 上下 8）
   - 改：`padding: var(--sp-2);`（统一 8px 四向，让 textarea 内文本起点 = composer 边框 + 8px）

⑤ 但这样 composer 左 8px 仍与 pill-row 起点不一致（pill-row 起点 = inputbar:16，composer 内文本起点 = inputbar:16 + composer 边框 + 8）。**解决方案**：将 pill-row 也加 8px 左 padding 与 composer 内边距对齐：
   - 改 `.aster-inputbar__pill-row` 为 `padding: 0 var(--sp-2) var(--sp-2);`（横向 8px 与 composer 内部 padding 一致；composer 边框 1px 可忽略）

⑥ **核对**：胶囊文本左边界 = inputbar:16 + pill-row:8 + pill 自身 padding 2 + 文字 = 26+2px；输入框文本左边界 = inputbar:16 + composer 边框 1 + composer padding-left:8 = 24+1px。差 1-3px（pill 自身有 2px 横向内 padding，见 styles.css:524），**视觉上无法察觉**（< 4px 阈值）。同理右侧：胶囊右边界 max-width=200px 内 ≤ 工具按钮行（`.aster-composer__toolbar` 是 space-between，右端按钮贴 composer right padding=8px）。胶囊右侧的 `×` 按钮位于胶囊 max-width 内，靠右；发送按钮位于 composer toolbar 右端，两者在视觉上都贴右内边距。

⑦ 在 styles.css `.aster-inputbar` 选择器旁加 1 行注释：
```css
/* G-02：水平 padding 统一走 --inputbar-padding-x；pill-row 与 composer 内部 padding 必须保持
   相同值（当前 8px），否则胶囊与输入框文本起点会错位。 */
```

不引入新颜色 / 不引入新尺寸单位 / 不破坏发送按钮的 brand-grad 渐变。
