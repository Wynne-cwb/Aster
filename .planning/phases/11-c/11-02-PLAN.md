## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| executeReverse case 'batch_reverse' → adapter.executeBatchReverse | batch_reverse 的 ops 参数来自 OperationLog reverse.args（由 adapter 在执行时构建，非 LLM 直接控制）|
| subOps 透传 ToolResult → OperationLogEntry | subOps 仅在 in-memory 生存，不序列化到 chatStore / localStorage（见 RESEARCH.md Open Q3）|

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-W1-01 | Tampering | executeReverse batch_reverse ops 参数 | mitigate | reverse.args.ops 在 batch.ts execute 时由 adapter.executeBatch 返回值构建（adapter 规范化 address），不由 LLM 的 ops 数组直接透传；Wave 2/5 integration test 断言 Record 对象 |
| T-11-W1-02 | Denial of Service | case 'batch_reverse' 降级路径 for 循环 | mitigate | 降级路径仅在 executeBatchReverse 不存在时触发；ops 数组由 batch entry 的 subOps 组成（已经过 D-06 上限 20 校验），循环次数有界 |
| T-11-W1-03 | Information Disclosure | ToolResult.subOps 透传到 appendOperation | accept | subOps 仅进 in-memory OperationLogEntry（不序列化）；DiffLogPanel 只渲染 humanLabel（不暴露 reverse.args 给用户）|
</threat_model>
