# Quick Task 260531-l7v: W1 部分成功 batch 抹掉熔断器失败计数

## 问题（Phase 11 code review W1）

`batch.ts` 部分失败分支（L212-219）返回 `ok: !partialOk || completedSubOps.length > 0`
→ 部分成功（完成≥1 步但中途失败）返回 `ok: true`。`loop-helpers.ts:146`
`if (result.ok) breaker.recordSuccess(...)` → 熔断器把部分失败的 batch 当成功，
batch_write 反复部分失败时熔断器无法开路。

## 关键约束（不可违反）

`loop-helpers.ts:155` 的 `appendOperation`（undo 记录）gated on `result.reverse`，
**与 ok 无关**。所以**不能**为了熔断器把 ok 改 false——否则 LLM 把部分成功当全失败，
会重做已完成的步骤（dataPayload.failed 本意是让 LLM 从失败步继续）。
**走解耦方案**：ok 保持 true，另加独立信号 `partialFailure` 给熔断器。

## Tasks

### Task 1 — 解耦实现
- files: `src/agent/tools/index.ts`, `src/agent/tools/write/batch.ts`, `src/agent/loop-helpers.ts`
- action:
  1. `ToolResult` 接口新增可选字段 `partialFailure?: boolean`（含注释：部分失败 batch
     信号，供 loop-helpers 通知熔断器；不影响 ok / undo 记录）。
  2. `batch.ts` 部分失败分支（L212-219）：保留 `ok: true`，当 `failAtIndex !== undefined`
     时加 `partialFailure: true`。
  3. `loop-helpers.ts:146-147`：改为
     `if (result.ok && !result.partialFailure) breaker.recordSuccess(tc.name);`
     `else breaker.recordFailure(tc.name, result.error?.code ?? 'PARTIAL_BATCH_FAILURE');`
  4. **不动** `appendOperation`（L155-168）——已完成 subOp 的 undo 记录靠 `result.reverse` 仍在。
- verify: `npx tsc --noEmit` 通过。
- done: 部分失败 batch → ok 仍 true（undo 保留）+ partialFailure=true（熔断器感知）。
