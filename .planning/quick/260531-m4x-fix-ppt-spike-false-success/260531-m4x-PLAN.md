# Quick Task 260531-m4x — 修复 PPT spike 工具网页版「假成功」

## 背景

真机 UAT（Office for Web PowerPoint）发现 3 个 spike 工具「报成功 ✅ 但实际没生效」——违反诚实底线：
- `set_shape_text_alignment`（PPT-02）
- `set_slide_background`（PPT-08）
- `rotate_shape`（PPT-05，附带 humanLabel undefined bug）

## 根因（已用 @types/office-js 核实）

1. **对齐用错属性名**：`PptAdapter.setShapeTextAlignment/restoreShapeAlignment` 读写 `paragraphFormat.alignment`——
   Office.js `ParagraphFormat` 没有 `.alignment`，正确属性是 `paragraphFormat.horizontalAlignment`
   （枚举 `PowerPoint.ParagraphHorizontalAlignment`：Left/Center/Right/Justify/…）。写不存在的属性 = 静默无效 = 假成功。
2. **背景用错 API**：`SlideBackgroundFill` **没有** `setSolidColor`/`foregroundColor`/`clear`（这些在 `ShapeFill` 上）。
   正确 API：`fill.setSolidFill({ color })` 写、`fill.type`（→"Solid"）读、`fill.getSolidFillOrNullObject().color` 读色、
   `slide.background.reset()` 还原默认。当前代码 cast 成假类型调不存在方法 → try/catch 吞掉 → beforeColor=null → 仍报 ✅。
3. **旋转**：`shape.rotation: number` 真实存在可写（API 路径对）；只缺「写后回读验证」。
4. **共性 bug**：spike 降级只 try/catch 读 before-image，**没验证写进去到底生没生效** → 网页版静默 no-op 时仍报成功。
5. **rotate humanLabel**：真机日志「第 undefined 张…「undefined」旋转至 45°」——schema 用 camelCase（slideIndex/shapeId），
   但 sibling PPT 工具用 snake_case，LLM 易传 snake_case → label 取 camelCase 键得 undefined。

## 任务

### Task 1 — 对齐换正确属性 + 三工具写后回读验证（PptAdapter.ts）
- `setShapeTextAlignment`/`restoreShapeAlignment`：`paragraphFormat.alignment` → `paragraphFormat.horizontalAlignment`；
  加 alignment 值归一化（left/center/… → Left/Center/…）。
- 三个 forward 方法（setShapeTextAlignment / rotateShape / setSlideBackground）写入 + sync 后**回读目标属性**与意图值归一化比对：
  - 一致 → `effective: true`，返回真实 before-image 供 undo。
  - 不一致（网页版 no-op）→ `effective: false`（诚实失败信号）。
- 背景：write 改 `setSolidFill({color})`，回读 `fill.type==='Solid'` 判定生效；before-image 经 `getSolidFillOrNullObject()` 读旧纯色；
  `restoreSlideBackground` 的 `setSolidColor`→`setSolidFill`、`clear()`→`background.reset()`。
- 旋转：回读 `shape.rotation` 数值比对（容差 0.5，含 360 环绕）。
- **files:** `src/adapters/PptAdapter.ts`
- **verify:** `npm test -- PptAdapter` 通过；类型 build 通过。
- **done:** 三 forward 方法返回显式 `effective` 信号；对齐/背景用正确 API。

### Task 2 — 工具层诚实失败 + rotate label 修复（ppt.ts）
- `setShapeTextAlignmentTool` / `rotateShapeTool` / `setSlideBackgroundTool`：`!effective` → 返回 `{ ok:false, error }`
  （文案：「此操作在网页版 PowerPoint 未生效（可能仅桌面版 PowerPoint 支持）」），**不带 reverse、不带 postState**
  → loop-helpers 不记 undo、UI 不报 ✅。`effective` 时正常返回真实 reverse/postState。
- rotate（及同族 camelCase 工具）humanLabel + execute 容错读 `slideIndex ?? slide_index`、`shapeId ?? shape_id`。
- **files:** `src/agent/tools/write/ppt.ts`
- **verify:** `npm test -- ppt.test` 通过。
- **done:** 假成功被 `!effective` 拦成诚实失败；rotate label 显示正确页号/形状 id。
