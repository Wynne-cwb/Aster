## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Security] buildChatSection 缺少 redactKey 脱敏**
- **Found during:** Task 2 测试执行（[STEPLOG REDACT] 断言失败）
- **Issue:** `buildChatSection` 直接输出 `msg.content` 到聊天记录节，未调 `redactKey()`。当用户把 sk-* Key 粘进了聊天消息，聊天记录节（原 5 节之一）会把 Key 原文带入报告。
- **Fix:** `buildDebugReport` 的 `import` 追加 `redactKey`；`buildChatSection` 对 `content` 调 `redactKey(rawContent)` 后输出。
- **Files modified:** `src/lib/debugReport.ts`
- **Commit:** `951ff66`

**2. [Rule 1 - Bug] clipboard.test.ts execCommand mock 用 `vi.spyOn` 报错**
- **Found during:** Task 2 首次运行（CLIPBOARD FALLBACK / DOUBLE FAIL 失败）
- **Issue:** jsdom 环境中 `document.execCommand` 不存在，`vi.spyOn(document, 'execCommand')` 抛 `execCommand does not exist`。
- **Fix:** 改用 `Object.defineProperty` 直接在 `document` 上定义 mock 函数，绕过 spyOn 的存在性检查。
- **Files modified:** `src/lib/clipboard.test.ts`
- **Commit:** `951ff66`

**3. [Rule 1 - Bug] debugReport.test.ts 新断言 `vi.fn() as typeof orig` 类型转换错误**
- **Found during:** Task 2 全量 tsc（3 个 error TS2352）
- **Issue:** 新增 3 个 `[STEPLOG *]` 测试用 `as typeof orig` 转换 partial mock，tsc strict 模式下报 ChatState 方法缺失。
- **Fix:** 改为 `as unknown as typeof orig`（与 copyStepLog.test.ts 现有写法一致）。
- **Files modified:** `src/lib/debugReport.test.ts`
- **Commit:** `951ff66`
