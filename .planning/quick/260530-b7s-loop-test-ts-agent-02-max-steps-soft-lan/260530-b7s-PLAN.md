关键事实（已 root-cause 确认，无需重查）：

**熔断器触发机制**（src/agent/circuit-breaker.ts）：
- WINDOW = 5，THRESHOLD = 3
- 同一 toolName 在窗口内 NOT_FOUND ≥ 3 次 → isOpen() = true
- loop-helpers.ts 在每次 dispatchTool 返回 NOT_FOUND 后调用 recordFailure(name, 'NOT_FOUND')
- 第 4 轮 isOpen('nonexistent') = true → abort('circuit') → status='idle'，loop return

**修复思路**：mock 每轮 yield 不同工具名（missing_tool_0…missing_tool_19）。
每个名字只累积 1 次 NOT_FOUND，不达 THRESHOLD，熔断器不触发，循环跑满 20 步，
pushSoftLanding 触发，agentStatus = 'soft-landing'，controller 未 abort。

**module-global 泄漏**：circuit-breaker.ts 暴露 __reset()（"仅供 vitest beforeEach 使用"）；
当前 beforeEach 未调用，需补上以保证跨测试隔离。
