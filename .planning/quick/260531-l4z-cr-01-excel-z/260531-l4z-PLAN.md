# Quick Task 260531-l4z: CR-01 Excel 列索引 >Z 非法地址修复

## 问题（Phase 10 code review CR-01）

`src/adapters/ExcelAdapter.ts` 的 `setColumnRowSize`（前向，L689）与
`restoreColumnRowSize`（inverse，L748）用 `String.fromCharCode(65 + idx)` 生成列地址。
当列索引 `idx >= 26`（即列 AA 起）时，`65 + 26 = 91 = '['`，生成非法 A1 列字母 →
非法 range 地址（如 `[:[`），Excel.run 会抛错或写错列。

## 方案

抽一个模块私有 helper `columnIndexToLetter(idx: number): string`（与
`normalizeText`/`buildTableFingerprint` 同级私有函数风格），把 0-based 列索引转成
多字母 A1 列字母（bijective base-26）：
- 0 → 'A'，25 → 'Z'，26 → 'AA'，27 → 'AB'，701 → 'ZZ'，702 → 'AAA'

`setColumnRowSize` 与 `restoreColumnRowSize` 两处列地址生成均改用此 helper。

保持现有安全性不变：读 before-image 失败仍对称（不动 try/catch 结构与 two-sync 流程）。
