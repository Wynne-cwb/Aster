# Quick Task 260527-o8j: Fix empty Lingui zh-CN catalog

## Problem
Task Pane rendered Lingui auto-generated hash ids (1HgTuA / bhewSJ / lsstWU /
AjjbWw / yFzc9v) instead of the zh-CN copy contract. Surfaced during Phase 1
human UAT (Test 1) in PowerPoint for Web after clearing the Office add-in cache.

## Root cause (verified)
- `src/i18n/index.ts` imports `./locales/zh-CN/messages` (bare). TS/Vite resolved
  this to a hand-stubbed empty placeholder `messages.ts` (`export const messages = {}`).
- The real compiled catalog landed in `messages.mjs` (lingui `compileNamespace: 'es'`),
  which the bare import never picked up.
- `package.json` `"build": "vite build"` never ran `lingui compile`, so even the
  compiled artifact was stale/absent in CI.
→ runtime `i18n.loadAndActivate` activated an empty catalog → macros render hash ids.

## Tasks
1. Switch `compile` to `lingui compile --typescript` so the catalog is generated
   into `messages.ts` (the path the bare import resolves to), typed via `Messages`.
2. Make `build` run `lingui compile --typescript && vite build` to prevent drift.
3. Commit `messages.po` (source catalog) so CI/Pages `lingui compile` has input.
4. Remove the stray `messages.mjs`.
