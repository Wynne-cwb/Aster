## Overview

v2.1「从能用到好用」是建立在 v2.0「Office 智能代理」基座上的**深化 + 打磨型 milestone（无架构 pivot）**。目标：让 Aster 从「能用」走到「好用」——agent 更懂三个宿主（per-host domain prompt）、能改更多东西（Word/Excel/PPT write tool 全补）、改得更快更准（批量操作 + Word 选区精度）、体验更顺手（UI 打磨套件）、记得住历史（聊天记录持久化）。多模态（看图/生图/文件/图库）拆到独立的 v2.2 milestone。

六大能力域 A–F：
- **A 能力变聪明** — per-host 系统 prompt 深化 + 用户偏好注入（含 prompt-injection 防御）
- **B 能力补全** — Word 5 / Excel 10 / PPT 8 共 23 个 write tool（triage 后高频痛点，参数化合并）
- **C 批量操作** — batch_write 单闭包单 sync + 一键 undo 整批
- **D Word 选区精度** — paragraphIndex + uniqueLocalId 精确定位多个相同文本
- **E UI 打磨** — XSS 防御 + 思考气泡 + DiffLog 跟随 loop + 表格边框 + 读卡降权 + 骨架屏
- **F 聊天记录持久化** — localStorage + 清空 + 20 轮截断 + 分文档 docKey

**贯穿约束（继续硬守）：** 无后台 / 纯 BYO Key / 纯浏览器直连 / 三宿主 Office.js API 子集 / 初始 bundle CI gate ≤82 KB gzip / P95 ≤10s / Key 不上传 / 0 净新增运行时依赖。
**项目原则确立（v2.1）：** AI 生成质量 >> token 成本 & 包体积——NFR-07 由 `<3000 字符硬 CI gate` 降为软提醒、NFR-08 去掉 toolDefs ≤15KB token 门；但 undo 守门、bundle gate、P95 仍硬卡。
