# Milestone v2.2: 多模态四件套

**Status:** ✅ SHIPPED 2026-06-03（tag `v2.2`，线上 `0d5fccf`）
**Phases:** 14–19（6 phases）
**Total Plans:** 25（14:6 · 15:5 · 16:5 · 17:6 · 18:3；Phase 19 = UAT/Release 无独立 plan）
**Core Value:** 在原生 Office 内部，让中文职场用户用自带 API Key 享受 **AI 代理** 能力，能完成绝大部分文档工作（多步任务、精细化操作、多模态）；无后台、BYO Key、纯浏览器直连。

## Overview

v2.2「多模态四件套」给 v2.0/v2.1 的 Office 智能代理加上**「看 / 读文件 / 生图 / 找图」四种多模态能力**。Provider 客户端（`aihubmix-vision.ts` / `aihubmix-image.ts`）已在基座里，但从未接进 agent loop / 无 tool / 无 UI；v2.2 把它们接进 agent loop、配 tool、配 UI，并顺手清掉 v2.1 遗留的 PPT casing 技术债。

四件套 + Provider 重写：
- **MDL AiHubMix Provider 重写** — 三生图模型三套 wire format（doubao URL / gpt-image-2 b64_json / gemini inlineData）+ 两套鉴权（Bearer / x-goog-api-key）统一为裸 base64；视觉/生图 model 清单修正；PPT 工具 casing 中央归一化根治（清 v2.1 技术债）
- **VIS 视觉看图** — agent「看」选中图（`get_shape_image` 第 12 个 read tool）+ 用户上传/粘贴图（回形针，FILE-06 前移）→ 都走 aihubmix-vision 返回文本 evidence
- **IMG 图片生成插入** — PPT/Word「生成图并插入」write tool，AI loop 内**自动直插**（设计反转）+ 只读结果卡 + model 可选
- **FILE 文件上传解析** — docx/xlsx/pdf/pptx 懒加载解析为文本注入 agent context；明确「附件」vs「agent 自取当前文档」UX 边界
- **LIB 公开图库检索** — Pexels BYO key 检索免费正版图并插入（复用 IMG insert helper）+ 摄影师署名

**贯穿约束（继续硬守）：** 无后台 / 纯 BYO Key / 纯浏览器直连 / 三宿主 Office.js API 子集 / 初始 bundle CI gate ≤82 KB gzip（解析等重模块懒加载）/ P95 ≤10s / Key 不上传 / **0 净新增运行时依赖**（mammoth/SheetJS/pdfjs/jszip 全懒加载，不进初始包）。
**项目原则延续（v2.1 确立）：** AI 生成质量 >> token 成本 & 包体积；undo 守门、bundle gate、P95 仍硬卡。
